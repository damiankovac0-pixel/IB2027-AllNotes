# A1.1.3 CPU Vs GPU

CPUs and GPUs are both processors, but they are designed for different kinds of work.

## Main Difference

A CPU is a general-purpose processor optimized for flexible, complex, sequential tasks.

A GPU is a specialized processor optimized for high-throughput parallel tasks.

## CPU Strengths

CPUs are good at:

- running the operating system
- managing applications
- handling user input
- making complex decisions quickly
- executing varied instructions
- multitasking between different types of work

CPUs usually have fewer cores, but each core is powerful and flexible.

## GPU Strengths

GPUs are good at:

- graphics rendering
- image and video processing
- matrix and vector calculations
- deep learning
- highly parallel workloads

GPUs usually have many more cores than CPUs, but each core is simpler.

## Design Philosophy

| Feature | CPU | GPU |
| --- | --- | --- |
| Purpose | general-purpose processing | specialized parallel processing |
| Cores | fewer, more powerful | many, smaller |
| Best workload | varied and sequential | repeated operations over large data |
| Example | operating system tasks | rendering pixels or training neural networks |
| Strength | flexibility and low-latency decisions | high throughput |

## SIMD

GPUs are suited to SIMD: single instruction, multiple data.

This means the same instruction is applied to many data values at once.

Example: applying the same brightness adjustment to every pixel in an image.

## Power And Efficiency

CPUs and GPUs both use power-management techniques, but their efficiency depends on workload.

- A CPU is efficient for mixed, decision-heavy tasks.
- A GPU is efficient when a task can be massively parallelized.

Using a GPU for a task that cannot be parallelized may not help much.

## Coordination Between CPU And GPU

The CPU often coordinates the overall program and sends suitable parallel tasks to the GPU.

For example, in a game:

- CPU handles game logic
- GPU handles visual rendering

Programming tools such as CUDA and OpenCL can help divide work between CPU and GPU.

## Exam Pattern

To compare CPU and GPU, focus on purpose, core design, task type, parallelism, and examples. CPUs are versatile and sequential; GPUs are parallel and high-throughput.
