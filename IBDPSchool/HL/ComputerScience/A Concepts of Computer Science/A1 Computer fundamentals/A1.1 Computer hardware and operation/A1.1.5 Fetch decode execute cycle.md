# A1.1.5 Fetch Decode Execute Cycle

The fetch-decode-execute cycle is the repeated process a CPU uses to run instructions.

It is also called the instruction cycle.

## Overview

```text
fetch -> decode -> execute -> repeat
```

The CPU repeats this cycle for each instruction in a program.

## Fetch

In the fetch stage, the CPU gets the next instruction from memory.

Typical steps:

1. The [[A1.1.1 CPU components#Registers|program counter]] stores the address of the next instruction.
2. This address is copied to the MAR.
3. The address bus sends the address to memory.
4. The control bus sends a read signal.
5. The instruction returns on the data bus.
6. The instruction is stored in the instruction register.
7. The program counter increments to the next instruction.

## Decode

In the decode stage, the control unit interprets the instruction.

It identifies:

- opcode: the operation to perform
- operands: the data or values involved
- addressing mode: how to find the operands

The control unit prepares the CPU components needed for execution.

## Execute

In the execute stage, the CPU performs the operation.

Examples:

- arithmetic operation in the ALU
- logic comparison
- reading from memory
- writing to memory
- jumping to another instruction
- storing a result in a register

## Example

For an instruction that adds two numbers:

1. Fetch the instruction from memory.
2. Decode it as an addition operation.
3. Execute it using the ALU.
4. Store the result, often in the accumulator or another register.

## Exam Pattern

For a 4-mark answer, describe fetch, decode, and execute, and mention the role of registers/buses. Strong answers include PC, MAR, MDR, IR, control unit, ALU, address bus, data bus, and control bus where relevant.
