# A1.1.6 Pipelining In Multi-Core Architectures

Pipelining improves CPU performance by overlapping stages of instruction processing.

Instead of waiting for one instruction to finish completely before starting the next, different instructions can be at different stages at the same time.

## Assembly Line Idea

Think of an assembly line.

One worker washes, another rinses, another dries. Once the first item moves on, the first worker can start the next item.

The same idea applies to instructions.

## Pipeline Stages

A simple pipeline may include:
- fetch
- decode
- execute
- memory access
- write back

While one instruction is executing, another may be decoding, and another may be fetching.

## Why It Improves Performance

Pipelining reduces idle time.

Without pipelining, CPU components may sit unused while another stage is busy. With pipelining, more parts of the CPU are active at once.

An optimized pipeline can approach one completed instruction per cycle after the pipeline is full.

## Multi-Core Architectures

In a multi-core CPU, each core can have its own pipeline.

This means multiple cores can process instructions in parallel, and within each core, pipelining can overlap instruction stages.

## Cache Coherency

In multi-core systems, cores may have separate caches and may also share cache or RAM.

Cache coherency ensures that different cores see consistent data when multiple cached copies of the same memory value exist.

Without cache coherency, one core might use an outdated value while another core has already changed it.

## Limitations

Pipelining is not perfect.

Problems include:

- branch instructions disrupting the pipeline
- data dependencies, where one instruction needs the result of another
- resource conflicts
- pipeline stalls
- overhead in coordinating multiple cores

## Exam Pattern

Pipelining improves performance by overlapping instruction stages so multiple instructions are processed simultaneously at different stages. In multi-core architectures, several cores can run pipelines in parallel, increasing throughput, but coordination and cache coherency are needed.
