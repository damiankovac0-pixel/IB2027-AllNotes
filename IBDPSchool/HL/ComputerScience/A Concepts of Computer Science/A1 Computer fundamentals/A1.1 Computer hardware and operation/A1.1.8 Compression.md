# A1.1.8 Compression

Compression reduces the number of bits needed to represent data.

Main benefits:

- files take less storage space
- files transfer faster over networks
- cloud and backup storage costs can be reduced

There are two main types: [[#Lossless Compression|lossless]] and [[#Lossy Compression|lossy]].

## Lossless Compression

Lossless compression reduces file size without losing any original data.

After decompression, the original data is restored exactly.

Use lossless compression when exact reconstruction matters.

Examples:

- text files
- source code
- some image formats
- database records
- medical or legal documents

## Run Length Encoding

Run length encoding, or RLE, is a simple lossless compression method.

It replaces repeated values with the value and the number of repeats.

Example:

```text
AAAAABBBCC -> 5A3B2C
```

RLE works well when data has long repeated runs. It works poorly when data changes constantly.

## Lossy Compression

Lossy compression reduces file size by discarding some data.

The decompressed file is not exactly the same as the original, but the loss may be acceptable if humans barely notice it.

Use lossy compression for:

- images
- audio
- video

Examples:

- JPEG
- MP3
- some video codecs

## Transform Coding

Transform coding is a lossy compression method often used in image and audio compression.

Basic idea:

1. Convert data into a form that separates important and less important information.
2. Keep the most noticeable information.
3. Reduce or discard less noticeable details.
4. Encode the result more compactly.

For JPEG, image blocks can be transformed from pixel values into frequency components. Low-frequency data represents broad smooth areas. High-frequency data represents details, edges, and textures.

Lossy compression often removes some high-frequency detail because humans may notice it less.

## Lossless vs Lossy

| Feature | Lossless | Lossy |
| --- | --- | --- |
| Original restored exactly | yes | no |
| Compression ratio | usually lower | usually higher |
| Best for | text, code, important records | images, audio, video |
| Quality loss | none | possible |

## Exam Pattern

Compression reduces file size for storage and transfer. Lossless compression restores the original exactly, while lossy compression discards less important data to achieve smaller file sizes but may reduce quality.
