# A1.2.2 Binary Storage Of Data

All data in a computer is stored as binary, but different data types use different encoding methods.

## Integers

An integer is a whole number.

Computers usually store integers using a fixed number of bits.

Example: an 8-bit value always uses 8 bits, even if the number is small.

```text
4 = 00000100
```

## Unsigned Integers

Unsigned integers represent only non-negative values.

With `n` bits, the range is:

```text
0 to 2^n - 1
```

Example: 8-bit unsigned integers can represent:

```text
0 to 255
```

## Signed Integers And Two's Complement

Signed integers need to represent positive and negative numbers.

Two's complement is a common method.

In two's complement, the most significant bit helps indicate whether the value is negative.

Benefits:

- represents positive and negative integers
- makes binary arithmetic easier for hardware
- has only one representation of zero

You do not need to overcomplicate it at this stage, but remember that fixed-width storage means the same bit pattern can mean different things depending on whether it is interpreted as signed or unsigned.

## Strings And Characters

Characters are stored using character encoding systems.

Examples:

- [[A1.2.1 Principal methods of representing data#ASCII|ASCII]]
- [[A1.2.1 Principal methods of representing data#UTF-8|UTF-8]]

A string is a sequence of encoded characters.

Example: the word `CAT` is stored as the binary codes for `C`, `A`, and `T`.

## Images

Digital images are made of pixels.

Each pixel stores colour information.

Common representation: RGB.

RGB stores three colour components:

- red
- green
- blue

Each component is often stored using 8 bits, giving values from `0` to `255`.

Example:

```text
RGB(255, 0, 0) = full red, no green, no blue
```

Important image factors:

- resolution: number of pixels
- colour depth: number of bits used per pixel
- compression: how file size is reduced

Higher resolution and higher colour depth usually mean larger file size.

## Audio

Sound is originally analogue, meaning it varies continuously.

To store audio digitally, the computer samples the sound wave.

### Sampling Rate

Sampling rate is how many samples are taken per second.

Example: `44.1 kHz` means 44,100 samples per second.

Higher sampling rate can capture more detail but increases file size.

### Bit Depth

Bit depth is the number of bits used to store each sample.

Common values include 16-bit and 24-bit.

Higher bit depth gives more possible amplitude values and can improve audio quality, but increases file size.

### Channels

Mono uses one channel.

Stereo uses two channels, left and right.

Stereo roughly doubles the data compared with mono at the same sampling rate and bit depth.

### Audio File Size

Basic formula:

```text
file size in bits = sampling rate * bit depth * channels * duration in seconds
```

## Video

Video is a sequence of images called frames, usually with audio.

Important factors:

- resolution
- frame rate
- colour depth
- duration
- audio quality
- compression method

Higher resolution, higher frame rate, and longer duration increase file size.

## Video Compression

### Spatial Compression

Spatial compression reduces redundancy within each frame.

It looks for repeated or less important information inside a single image frame.

### Temporal Compression

Temporal compression reduces redundancy between frames.

Instead of storing every full frame, it stores changes between frames.

This is useful when much of the scene stays the same.

## Gray Code

Gray code is a binary system where consecutive values differ by only one bit.

It is useful in systems where transition errors matter, such as rotary encoders or robotic arm position sensors.

Benefit: because only one bit changes at a time, the risk of reading an incorrect intermediate value is reduced.

## Floating Point

Floating-point representation stores very large, very small, or fractional numbers.

It usually includes:

- sign bit
- exponent
- mantissa or significand

Floating point is useful for scientific computing, graphics, and engineering.

Limitation: not every decimal value can be represented exactly, so rounding errors can occur.

## Exam Pattern

For binary storage questions, explain the data type first, then the binary representation method. For audio, mention sampling rate, bit depth, and channels. For images, mention pixels, resolution, colour depth, and RGB. For video, mention frames and compression.
