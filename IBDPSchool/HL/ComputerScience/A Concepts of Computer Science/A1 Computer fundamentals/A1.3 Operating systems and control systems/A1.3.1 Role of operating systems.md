# A1.3.1 Role Of Operating Systems

An operating system, or OS, is system software that acts as an interface between users, applications, and hardware.

Examples:

- Windows
- macOS
- Linux
- Android
- iOS
- Unix
- BSD

## Core Role

The OS manages system resources and provides services that allow software to run without needing to directly control hardware.

Resources managed by the OS include:

- CPU time
- memory
- storage
- input/output devices
- network connections
- files
- security permissions

## Abstraction

Abstraction means hiding complex details behind a simpler interface.

The OS abstracts hardware complexity so users and applications do not need to manage low-level details directly.

Example: when you open a game, you do not manually allocate memory, schedule threads, manage interrupts, or communicate with the GPU. The OS helps coordinate those tasks.

## Interface Between User And Hardware

The OS provides ways to interact with the system:

- graphical user interface
- command-line interface
- file explorer
- settings tools
- application launcher

## Interface Between Applications And Hardware

Applications use OS services to access hardware safely.

Example: an application asks the OS to save a file. The application does not directly control the disk; the OS handles the file system and storage device.

## Why There Is No Single Best OS

Different operating systems are designed for different purposes.

Examples:

- embedded OS for small devices
- mobile OS for phones
- desktop OS for general users
- server OS for reliability and networking
- real-time OS for predictable timing

The best OS depends on the purpose, hardware, security requirements, usability, and performance needs.
