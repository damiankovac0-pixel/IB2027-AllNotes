# A1.3.3 Scheduling Approaches

Scheduling is how the operating system decides which process gets CPU time and for how long.

The aim is to keep the system responsive, fair, and efficient.

## Key Terms

| Term | Meaning |
| --- | --- |
| Throughput | number of processes completed in a time period |
| Turnaround time | time from process arrival to completion |
| Waiting time | time spent waiting in the ready queue |
| CPU utilization | percentage of time the CPU is busy |
| Time slice or quantum | fixed amount of CPU time given to a process |
| Pre-emption | interrupting a running process to run another |
| Starvation | a process waits indefinitely for resources |
| Context switching | saving one process state and loading another |

## First Come First Served

First come first served, or FCFS, runs processes in the order they arrive.

Advantages:

- simple
- predictable
- low overhead

Disadvantages:

- long processes can delay shorter ones
- poor responsiveness in interactive systems
- not ideal for multi-user systems

Best for simple batch systems where process lengths are predictable.

## Round Robin

Round robin gives each process a fixed time quantum, then moves it to the back of the queue if it has not finished.

Advantages:

- fairer than FCFS
- good for time-sharing systems
- improves responsiveness

Disadvantages:

- too small a quantum causes excessive context switching
- too large a quantum behaves more like FCFS
- overhead from frequent switching

Best for multitasking and interactive systems.

## Priority Scheduling

Priority scheduling gives CPU time to the process with the highest priority.

Priorities can be:

- static: set when process is created
- dynamic: can change over time

Advantages:

- important tasks run sooner
- useful in real-time or critical systems

Disadvantages:

- low-priority processes may starve
- priority assignment can be difficult

Ageing can reduce starvation by gradually increasing the priority of waiting processes.

## Multilevel Queue Scheduling

Multilevel queue scheduling separates processes into different queues based on type or priority.

Example queues:

- system processes
- interactive user processes
- background batch jobs

Each queue may use a different scheduling algorithm.

Advantages:

- handles different process types well
- allows important queues to be prioritized
- suitable for complex operating systems

Disadvantages:

- more complex
- lower queues may suffer starvation
- queue design affects fairness

## Context Switching

Context switching happens when the OS saves the state of one process and loads another.

Saved state may include:

- register values
- program counter
- memory pointers
- process status

Context switching creates the illusion that many processes run at the same time, but too much switching reduces performance.

## Comparison

| Algorithm | Main strength | Main weakness |
| --- | --- | --- |
| FCFS | simple and predictable | poor response if long jobs arrive first |
| Round robin | fair and responsive | quantum choice affects overhead |
| Priority | important tasks run first | starvation risk |
| Multilevel queue | handles different process classes | more complex and can be unfair |

## Exam Pattern

When comparing scheduling algorithms, discuss fairness, starvation, overhead, responsiveness, and suitable use cases.
