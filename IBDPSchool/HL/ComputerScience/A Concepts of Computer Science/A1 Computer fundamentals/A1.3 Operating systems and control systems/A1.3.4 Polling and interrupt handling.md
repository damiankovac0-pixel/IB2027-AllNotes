# A1.3.4 Polling And Interrupt Handling

Polling and interrupt handling are two methods for managing communication between the CPU and devices.

## Interrupt Handling

An interrupt is a signal that tells the CPU a device or program needs attention.

Example: a keyboard key is pressed, a mouse moves, or network data arrives.

Basic process:

1. Device sends an interrupt signal.
2. CPU pauses its current task.
3. CPU saves its current state.
4. Interrupt service routine, or ISR, runs.
5. The event is handled.
6. CPU restores its previous state and continues.

## Interrupt Service Routine

An ISR is a special routine that handles a specific interrupt.

Example: for a mouse movement interrupt, the ISR reads movement data, calculates the new cursor position, and triggers the screen update.

## Polling

Polling is when the CPU repeatedly checks a device to see whether it needs attention.

Example: the CPU checks a sensor every second to see whether a value has changed.

## Interrupts vs Polling

| Factor | Interrupts | Polling |
| --- | --- | --- |
| CPU use | CPU can do other work until event occurs | CPU spends time checking repeatedly |
| Responsiveness | usually high | depends on polling interval |
| Simplicity | more complex to implement | simpler to implement |
| Power use | often better for battery devices | can waste power if checks are frequent |
| Best for | unpredictable or urgent events | simple, predictable, low-frequency checks |

## When Interrupts Are Better

Interrupts are usually better when:

- events are unpredictable
- fast response is needed
- CPU should not waste cycles checking
- device is battery powered
- network or user input must be handled quickly

Example: network packets arriving during a video call should be handled quickly to reduce latency.

## When Polling Is Better

Polling may be better when:

- events are predictable
- timing must be controlled
- the system is simple
- immediate response is not critical
- the device is checked rarely

Example: a simple embedded sensor system may poll temperature every few seconds.

## Latency

Latency is the delay between an event and the system's response.

Interrupts usually reduce latency for unpredictable events, while polling latency depends on how often the system checks.

## Exam Pattern

To evaluate polling and interrupts, compare responsiveness, CPU overhead, power consumption, implementation complexity, event frequency, and whether the system needs low latency.
