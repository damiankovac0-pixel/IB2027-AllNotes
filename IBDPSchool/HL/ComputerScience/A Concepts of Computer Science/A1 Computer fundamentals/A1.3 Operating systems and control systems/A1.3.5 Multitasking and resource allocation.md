# A1.3.5 Multitasking And Resource Allocation

Multitasking allows a computer to run multiple tasks seemingly at the same time.

The OS achieves this by rapidly switching between processes and allocating resources such as CPU time, memory, files, and I/O devices.

## Multitasking

Multitasking depends on:

- scheduling
- context switching
- memory management
- resource allocation
- synchronization

The user experiences several programs running at once, even if the CPU is switching between them very quickly.

## Resource Allocation

Resource allocation is how the OS decides which process gets which resource.

Resources include:

- CPU time
- RAM
- storage access
- printers
- network bandwidth
- files
- input/output devices

The OS must allocate resources efficiently and fairly.

## Paging

Paging divides memory into fixed-size blocks.

It helps the OS manage memory and supports virtual memory.

## Segmentation

Segmentation divides memory into logical sections such as code, data, and stack segments.

It helps organize and protect memory.

## Resource Contention

Resource contention occurs when multiple processes want the same limited resource at the same time.

Example: two processes trying to print to the same printer.

The OS must coordinate access to avoid conflict.

## Semaphores

A semaphore is a synchronization tool used to control access to shared resources.

Types:

- binary semaphore: allows one process at a time
- counting semaphore: allows a limited number of processes at a time

Example: a binary semaphore can ensure only one process writes to a file at once.

## Deadlock

Deadlock happens when processes are stuck waiting for each other to release resources.

Example:

- Process A holds Resource 1 and waits for Resource 2.
- Process B holds Resource 2 and waits for Resource 1.
- Neither can continue.

Ways to handle deadlock:

- prevention
- avoidance
- detection
- recovery by stopping processes or reallocating resources

## Livelock

Livelock happens when processes keep changing state in response to each other but make no progress.

Unlike deadlock, the processes are active, but they still do not complete useful work.

## Priority Inversion

Priority inversion occurs when a high-priority process is blocked because a lower-priority process holds a needed resource.

This can harm real-time performance.

## Challenges

The OS must balance:

- fairness
- performance
- responsiveness
- avoiding starvation
- avoiding deadlock
- minimizing context-switching overhead
- maintaining data consistency

## Exam Pattern

For multitasking questions, explain scheduling and context switching. For resource allocation questions, mention contention, semaphores, deadlock, starvation, and how the OS prevents processes interfering with each other.
