# A2.1.4 Transport And Application Protocols

A protocol is a set of rules that defines how data is structured, transmitted, received, and interpreted across a network.

Protocols define things such as:

- message format
- timing
- sequencing
- error checking
- connection setup and termination

## Stateful And Stateless Protocols

A stateful protocol remembers information about a communication session.

A stateless protocol treats each request independently.

HTTP is stateless by design, but websites can maintain state using cookies and sessions.

## TCP

Transmission Control Protocol, or TCP, is a transport-layer protocol focused on reliability.

Key features:

- connection establishment using a handshake
- ordered delivery
- acknowledgements
- retransmission of lost data
- error checking
- flow control

Use TCP when correctness and order matter.

Examples:

- web page loading
- email
- file transfer

## UDP

User Datagram Protocol, or UDP, is a transport-layer protocol focused on speed and low overhead.

Key features:

- connectionless
- no guarantee of delivery
- no guarantee of order
- lower overhead than TCP
- useful when speed matters more than perfect reliability

Use UDP when small delays are worse than occasional loss.

Examples:

- live video/audio streaming
- online gaming
- voice calls
- DNS queries in many cases

## TCP vs UDP

| Feature | TCP | UDP |
| --- | --- | --- |
| Connection | connection-oriented | connectionless |
| Reliability | reliable | not guaranteed |
| Ordering | preserves order | not guaranteed |
| Speed | slower due to overhead | faster and lighter |
| Best for | accuracy and completeness | real-time speed |

## HTTP

Hypertext Transfer Protocol, or HTTP, is an application-layer protocol used for web communication.

It handles requests and responses between clients and web servers.

Example:

- browser sends an HTTP GET request
- server returns a web page or resource

HTTP is stateless, so each request is independent unless state is managed separately.

## Cookies And Sessions

Cookies are small files stored on the client device.

They can store state information such as login status, preferences, or tracking identifiers.

Sessions allow a web application to maintain user-specific data across multiple HTTP requests.

## HTTPS

HTTPS is HTTP with security provided by TLS.

It protects communication by encrypting data between browser and server.

Benefits:

- confidentiality: data is harder to read if intercepted
- integrity: data is protected from undetected tampering
- authentication: helps verify the server identity using certificates

## DNS

Domain Name System, or DNS, converts domain names into IP addresses.

Example:

```text
www.example.com -> IP address
```

Basic DNS lookup:

1. Browser needs the IP address for a domain.
2. Request goes to a local DNS server.
3. If cached, the IP address is returned.
4. If not cached, queries may go to root, top-level domain, and authoritative DNS servers.
5. The IP address is returned and may be cached for future use.

## DHCP

Dynamic Host Configuration Protocol, or DHCP, automatically gives devices network configuration.

DHCP can provide:

- IP address
- subnet mask
- default gateway
- DNS server address

Dynamic allocation gives an address for a lease period. Static allocation reserves an address for a device, often based on MAC address.

## Exam Pattern

For protocol questions, state what layer/type it belongs to and what problem it solves. TCP is reliable and ordered; UDP is fast and connectionless; HTTP transfers web resources; HTTPS encrypts HTTP; DNS resolves names; DHCP assigns network configuration.
