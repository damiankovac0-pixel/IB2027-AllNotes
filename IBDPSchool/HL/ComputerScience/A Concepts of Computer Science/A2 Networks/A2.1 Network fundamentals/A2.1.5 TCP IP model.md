# A2.1.5 TCP IP Model

The TCP/IP model is a layered model for understanding network communication.

Each layer has a specific role. Data moves down the layers when being sent and up the layers when being received.

## Layers

```text
Application
Transport
Internet
Network interface
```

## Application Layer

The application layer provides protocols used by applications.

Examples:

- HTTP and HTTPS for web browsing
- SMTP for email sending
- POP3/IMAP for email retrieval
- DNS for domain name resolution
- DHCP for network configuration

This layer forms the application-level message.

## Transport Layer

The transport layer provides communication between processes on different hosts.

Main protocols:

- [[A2.1.4 Transport and application protocols#TCP|TCP]]
- [[A2.1.4 Transport and application protocols#UDP|UDP]]

TCP provides reliability using sequencing, acknowledgements, checksums, and retransmission.

UDP provides lower overhead and faster connectionless communication.

## Internet Layer

The internet layer handles addressing and routing between networks.

Main protocol:

- IP

This layer adds source and destination IP addresses and determines where packets should go next.

Important concepts:

- IP address
- packet
- routing
- next hop
- time to live, or TTL

## Network Interface Layer

The network interface layer handles communication over the local physical or wireless network medium.

It deals with:

- Ethernet frames
- Wi-Fi frames
- MAC addresses
- physical transmission of bits
- conversion to electrical, optical, or radio signals

## Sending A Web Request

Example: a browser requests a web page.

1. Application layer creates an HTTP or HTTPS request.
2. Transport layer wraps it in a TCP segment and may establish a connection.
3. Internet layer wraps it in an IP packet with source and destination IP addresses.
4. Network interface layer wraps it in a frame and transmits it over the local network.

## Receiving A Web Response

On the receiving device, the process reverses:

1. Network interface layer receives the frame.
2. Internet layer checks the IP packet.
3. Transport layer reorders and checks TCP segments if needed.
4. Application layer receives the HTTP/HTTPS data.

## Encapsulation

Encapsulation is the process of adding layer-specific headers as data moves down the model.

Each layer adds information needed for its job.

## Why Layers Are Useful

Layering helps because:

- each layer has a clear responsibility
- protocols can be changed at one layer without redesigning everything
- troubleshooting is easier
- different technologies can work together
- complex communication becomes easier to understand

## Exam Pattern

For TCP/IP model questions, name each layer and describe its role. For longer answers, explain encapsulation and how data moves down layers when sent and up layers when received.
