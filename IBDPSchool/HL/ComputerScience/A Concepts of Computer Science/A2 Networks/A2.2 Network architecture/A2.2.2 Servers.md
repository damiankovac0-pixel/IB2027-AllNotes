# A2.2.2 Servers

A server provides services, resources, or data to client devices over a network.

Different server types solve different network problems.

## DNS Server

A DNS server translates domain names into IP addresses.

Example:

```text
www.example.com -> IP address
```

Purpose:

- makes the web easier for humans to use
- avoids needing to memorize IP addresses
- supports distributed name resolution

Reliability matters because if DNS fails, users may not be able to reach services even if the services themselves are online.

This connects to [[A2.1.4 Transport and application protocols#DNS|DNS]].

## DHCP Server

A DHCP server automatically assigns network configuration to devices.

It can provide:

- IP address
- subnet mask
- default gateway
- DNS server addresses

## DORA Process

DHCP commonly uses DORA:

1. Discovery: client broadcasts to find DHCP servers.
2. Offer: DHCP server offers an IP address.
3. Request: client requests one offered address.
4. Acknowledgment: server confirms the lease.

Benefits:

- reduces manual configuration
- avoids many IP address conflicts
- useful for networks where devices frequently join and leave

## File Server

A file server stores and manages files for network users.

Functions:

- central file storage
- file sharing
- permission management
- backups
- mounted network drives

Common protocols include SMB, NFS, and AFP.

Important issues:

- scalability: more storage and faster network cards may be needed
- reliability: RAID can help protect against disk failure
- security: permissions must be managed carefully

## Mail Server

A mail server sends, receives, and stores email.

Functions:

- receive incoming mail
- send outgoing mail
- store mailboxes
- filter spam or malicious content
- support mail clients

Common protocols may include SMTP, POP3, and IMAP.

## Proxy Server

A proxy server acts as an intermediary between a client and another server.

Client-side proxy uses:

- filter content
- cache web resources
- hide internal client details
- enforce access policies
- improve performance by serving cached content

Reverse proxy uses:

- acts on behalf of servers
- distributes incoming requests
- caches common responses
- improves scalability and fault tolerance
- hides internal server structure

## Web Server

A web server hosts and serves web pages or web applications.

It can serve:

- static files such as HTML, CSS, images
- dynamic responses generated by applications

Examples of web server software include Nginx and Apache.

Important issues:

- scalability for high traffic
- security against attacks
- reliability and uptime
- load balancing

## Server Comparison

| Server | Main function |
| --- | --- |
| DNS | domain name to IP address |
| DHCP | automatic network configuration |
| File | shared file storage |
| Mail | email sending, receiving, storing |
| Proxy | intermediary, filtering, caching, privacy |
| Web | hosts web content and applications |

## Exam Pattern

For server questions, identify the server type and describe the service it provides. For higher marks, include scalability, reliability, and security considerations.
