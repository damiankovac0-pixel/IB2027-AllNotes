# A2.2.3 Networking Models

Networking models describe how devices provide and access resources on a network.

The two main models are client-server and peer-to-peer.

## Client-Server Model

In the client-server model, clients request services and servers provide them.

Examples:

- browser requests web page from web server
- email client uses a mail server
- banking app connects to bank servers
- office computer accesses file server

Advantages:

- centralized management
- easier to enforce security policies
- easier backup and data consistency
- servers can be optimized for their role
- suitable for large organizations and sensitive systems

Disadvantages:

- server can be a single point of failure
- server is a high-value target for attacks
- scaling may require expensive infrastructure
- server maintenance is required

Best for:

- online banking
- enterprise systems
- email services
- controlled file storage
- web applications

## Peer-To-Peer Model

In a peer-to-peer model, each device can act as both client and server.

There is no single central server coordinating all resources.

Examples:

- BitTorrent file sharing
- some voice/video call systems
- blockchain networks

Advantages:

- no single central point of failure
- can be cost-effective
- resources are distributed
- can scale as more peers join

Disadvantages:

- harder to manage centrally
- security policies are harder to enforce
- performance depends on peers being available
- data consistency can be difficult
- less suitable for sensitive centralized services like banking

## Client-Server vs Peer-To-Peer

| Factor | Client-server | Peer-to-peer |
| --- | --- | --- |
| Control | centralized | decentralized |
| Resource location | mainly server | distributed across peers |
| Security management | easier centrally | harder to enforce |
| Failure risk | server failure can affect all | no single central failure point |
| Cost | server infrastructure needed | uses peer resources |
| Good for | banking, email, web apps | file sharing, blockchain |

## Why Online Banking Uses Client-Server

Online banking needs strong authentication, data consistency, centralized transaction processing, logging, and security controls.

A peer-to-peer model would be inappropriate because it would be harder to guarantee trust, consistency, and control over sensitive financial data.

## Exam Pattern

When comparing models, discuss centralization, control, scalability, reliability, security, cost, and suitability for the scenario.
