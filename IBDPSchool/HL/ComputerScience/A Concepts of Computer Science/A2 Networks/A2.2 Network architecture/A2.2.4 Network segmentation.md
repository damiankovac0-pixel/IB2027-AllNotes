# A2.2.4 Network Segmentation

Network segmentation divides a larger network into smaller logical parts.

The main purposes are:

- improve security
- reduce congestion
- improve performance
- simplify management
- apply different policies to different groups

## Why Segment A Network?

Example: in a bank, a guest Wi-Fi network should not access the customer-account network.

Segmentation limits which devices can communicate directly.

Benefits:

- contains security breaches
- separates sensitive systems
- reduces unnecessary broadcast traffic
- improves traffic management
- allows department-specific rules

## Subnetting

Subnetting divides a network into smaller networks using IP address ranges and subnet masks.

Example:

- staff devices: `192.168.0.x`
- student devices: `192.168.1.x`

Devices in the same subnet can usually communicate directly.

Devices in different subnets usually need a router.

## Subnet Mask

A subnet mask identifies which part of an IP address is the network portion and which part is the host portion.

Example:

```text
IP address: 192.168.72.5
Subnet mask: 255.255.255.0
Network portion: 192.168.72
Host portion: 5
```

Device `192.168.72.100` is on the same subnet.

Device `192.168.70.5` is on a different subnet.

## CIDR Notation

CIDR notation writes a subnet using a slash and the number of network bits.

Example:

```text
192.168.100.0/22
```

This tells how many bits are used for the network portion.

## VLAN

A VLAN, or virtual local area network, creates separate logical networks on the same physical switch infrastructure.

Example: one switch can have:

- ports 1-10 in VLAN 10 for Accounting
- ports 11-20 in VLAN 20 for Sales

Accounting devices can communicate with each other, and Sales devices can communicate with each other, but the two VLANs are separated unless routing is configured between them.

## VLAN Benefits

- separates departments without buying separate switches
- reduces broadcast traffic
- improves security
- allows different policies for different groups
- flexible because VLAN membership can be changed by configuration

## Subnetting vs VLANs

| Concept | Main idea |
| --- | --- |
| Subnetting | separates networks using IP address ranges |
| VLAN | separates networks logically at switch level |

They are often used together: each VLAN may have its own subnet.

## Security Limits

Segmentation improves security, but it is not complete protection by itself.

It should be combined with:

- firewall rules
- access control
- monitoring
- authentication
- patching
- least privilege permissions

## Exam Pattern

For segmentation questions, explain that it divides a network into smaller logical parts to improve security, performance, and management. Use subnetting and VLANs as examples, and explain how they limit unnecessary or unauthorized traffic between groups.
