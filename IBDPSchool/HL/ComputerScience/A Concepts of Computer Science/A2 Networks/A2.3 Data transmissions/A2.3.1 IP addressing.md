# A2.3.1 IP Addressing

An IP address is a numerical address assigned to a device on a network that uses the Internet Protocol.

IP addresses allow packets to be routed from source to destination across networks.

## Internet Protocol

The Internet Protocol, or IP, handles addressing and routing of packets between networks.

IP sits at the [[A2.1.5 TCP IP model#Internet Layer|internet layer]] of the TCP/IP model.

## IPv4

IPv4 uses 32-bit addresses.

It is usually written as four decimal numbers separated by dots.

Example:

```text
192.168.1.1
```

Each section is one byte, so each number must be from `0` to `255`.

IPv4 provides about 4.3 billion possible addresses, but many are reserved and the public address space has become limited.

## IPv6

IPv6 uses 128-bit addresses.

It is usually written as eight groups of hexadecimal digits separated by colons.

Example:

```text
2001:0db8:85a3:0000:0000:8a2e:0370:7334
```

IPv6 was designed largely because IPv4 address space was not enough for the growth of the internet.

Advantages of IPv6:

- vastly larger address space
- better support for modern internet growth
- reduced need for NAT in many designs
- improved address planning and scalability

## IPv4 vs IPv6

| Feature | IPv4 | IPv6 |
| --- | --- | --- |
| Address length | 32 bits | 128 bits |
| Format | dotted decimal | colon-separated hexadecimal |
| Example | `192.168.1.1` | `2001:0db8:85a3::8a2e:0370:7334` |
| Address space | about 4.3 billion | extremely large |
| Main issue solved | original internet addressing | IPv4 address exhaustion |

## Public IP Addresses

A public IP address is globally unique and routable on the public internet.

Used by:

- websites
- public servers
- routers connecting networks to the internet
- devices that must be reachable from outside the local network

Public IP addresses are managed through global and regional allocation systems so that no two public internet devices use the same address at the same time.

## Private IP Addresses

A private IP address is used inside a private network and is not directly routable on the public internet.

Common IPv4 private ranges:

```text
10.0.0.0 to 10.255.255.255
172.16.0.0 to 172.31.255.255
192.168.0.0 to 192.168.255.255
```

Private addresses are common in:

- home networks
- school networks
- office LANs
- internal servers

Different private networks can reuse the same private IP ranges because those addresses are not directly visible on the global internet.

## Public vs Private IP Addresses

| Feature | Public IP | Private IP |
| --- | --- | --- |
| Internet visibility | routable globally | not directly routable |
| Uniqueness | globally unique | only unique inside local network |
| Common use | websites, internet-facing servers | home/office internal devices |
| Needs NAT for internet access | usually no | usually yes |

## Static IP Addresses

A static IP address stays the same each time the device connects.

Best for:

- web servers
- email servers
- printers
- network devices
- services that must be found at a stable address

Advantages:

- predictable
- easier for hosting services
- simpler for remote access and firewall rules

Disadvantages:

- requires management
- can create conflicts if configured incorrectly
- less flexible for networks with many changing devices

## Dynamic IP Addresses

A dynamic IP address is assigned automatically and may change over time.

This is often done using [[A2.1.4 Transport and application protocols#DHCP|DHCP]].

Best for:

- home networks
- guest Wi-Fi
- schools
- offices with many changing devices

Advantages:

- easier administration
- efficient address reuse
- fewer manual configuration errors

Disadvantages:

- address may change
- less suitable for servers that need a stable address

## Network Address Translation

Network Address Translation, or NAT, allows devices using private IP addresses to access the public internet through a router's public IP address.

Basic process:

1. A private device sends a packet to the internet.
2. The NAT router replaces the private source IP with the router's public IP.
3. The router records the mapping in a NAT table, often using port numbers.
4. The internet server replies to the router's public IP.
5. The router uses its NAT table to forward the reply to the correct private device.

## Why NAT Matters

NAT helps because:

- many private devices can share one public IPv4 address
- it reduces demand for public IPv4 addresses
- internal private addresses are hidden from the public internet

Security note: NAT hides internal addresses, but it is not a complete security system. Firewalls and access controls are still needed.

## Loopback Address

The IPv4 loopback address is:

```text
127.0.0.1
```

It refers back to the same device.

IPv6 loopback is:

```text
::1
```

Loopback is used for testing network software on the local machine.

## Exam Pattern

Be ready to compare IPv4 and IPv6, public and private IP addresses, static and dynamic IP addresses, and explain how NAT lets many private devices share one public IP address.
