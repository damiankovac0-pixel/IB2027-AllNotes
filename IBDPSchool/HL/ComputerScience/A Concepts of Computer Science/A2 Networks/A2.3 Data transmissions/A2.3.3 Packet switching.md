# A2.3.3 Packet Switching

Packet switching sends data across a network by splitting it into small blocks called packets.

Each packet can travel independently across the network and may take a different path to the destination.

## Why Packet Switching Is Used

Packet switching improves efficiency because the network does not need to reserve one continuous path for an entire message.

Benefits:

- better use of network resources
- packets can route around congestion
- packets can route around failed links
- multiple users can share the same network infrastructure
- large messages can be broken into manageable pieces

## Packet

A packet is a small unit of transmitted data.

It usually contains:

- header
- payload
- error-checking information

## Payload

The payload is the actual data being carried.

Example: part of a file, image, video stream, web request, or message.

## Packet Header

The header contains control information needed to deliver and reconstruct the data.

Header information may include:

- source IP address
- destination IP address
- sequence number
- protocol information
- time to live, or TTL
- checksum or error-checking data

## Segmentation

Segmentation is splitting data into smaller packets before transmission.

Example: a large image is too big to send as one piece, so it is divided into many packets.

## Encapsulation

Encapsulation is adding protocol information around data as it moves down the [[A2.1.5 TCP IP model|TCP/IP model]].

Example:

```text
application data -> TCP segment -> IP packet -> Ethernet frame
```

Each layer adds its own header or trailer.

## Decapsulation

Decapsulation is removing protocol information as data moves up the TCP/IP layers at the receiving device.

The destination removes headers and reconstructs the original data.

## Routing Packets

Packets move through routers between networks.

Routers inspect the destination IP address and use routing tables to decide the next hop.

Packets from the same message may take different paths depending on:

- congestion
- route availability
- router decisions
- network failures
- routing protocol metrics

## Routing Table

A routing table is data stored in a router that helps it decide where to forward packets.

It may include:

- destination network
- subnet mask or prefix
- next hop
- outgoing interface
- route metric or cost

## Switches And Routers

Switches move data within a LAN using MAC addresses.

Routers move packets between networks using IP addresses.

This distinction is important:

- switch: local network forwarding
- router: inter-network routing

## Reassembly

When packets arrive, they may be out of order.

Sequence numbers allow the destination to reorder packets correctly and reconstruct the original message.

If packets are missing or corrupted, protocols such as [[A2.1.4 Transport and application protocols#TCP|TCP]] can request retransmission.

## Error Checking

Packets can include checksums or other error-detection information.

This helps the receiver detect whether data was corrupted during transmission.

## Packet Switching vs Circuit Switching

Packet switching does not reserve a dedicated path for the whole communication.

Circuit switching reserves a dedicated path for the duration of the communication.

Packet switching is more flexible and efficient for internet traffic, where many users share the same infrastructure.

## Exam Pattern

To explain packet switching, describe segmentation into packets, headers with addressing and sequence data, independent routing through routers, possible different paths, and reassembly at the destination.
