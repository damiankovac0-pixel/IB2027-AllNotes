# A2.3.4 Static And Dynamic Routing

Routing is the process of selecting paths for packets to travel across a network.

Routers use routing tables to decide where to forward packets next.

## Static Routing

Static routing uses manually configured routes.

A network administrator enters routes into the router configuration.

A static route usually includes:

- destination network
- subnet mask or prefix
- next hop address
- outgoing interface

## How Static Routing Works

1. A packet arrives at the router.
2. The router checks the destination IP address.
3. The router searches the routing table for a matching static route.
4. If a route matches, the packet is sent to the configured next hop or interface.
5. If no route matches, it may be dropped or sent to a default route if one exists.

## Static Routing Advantages

- predictable network behaviour
- simple for small networks
- low processing overhead
- no bandwidth used for routing updates
- can be more secure because routes do not automatically change

## Static Routing Disadvantages

- manual configuration can be time-consuming
- does not adapt automatically to network failures
- not scalable for large changing networks
- configuration mistakes can break connectivity

Best for:

- small stable networks
- simple LANs
- routes that rarely change
- default routes

## Dynamic Routing

Dynamic routing allows routers to automatically discover and update routes using routing protocols.

Routers exchange information about network topology and route availability.

Examples of routing protocols include RIP and other dynamic routing protocols.

## How Dynamic Routing Works

1. Routers share routing information with each other.
2. Each router updates its routing table.
3. If a link fails or a better route appears, routers adjust routes automatically.
4. Packets are forwarded based on current routing information.

## Dynamic Routing Advantages

- adapts to network changes
- better for large networks
- can route around failures
- reduces manual maintenance
- can choose routes based on metrics such as hop count, cost, bandwidth, or delay

## Dynamic Routing Disadvantages

- more complex to configure
- uses CPU, memory, and bandwidth for routing updates
- convergence time can affect performance
- routing protocol traffic can create security risks if not protected

## Convergence

Convergence is the time it takes for routers to update their routing tables and agree on valid routes after a network change.

During convergence, packets may be delayed, misrouted, or dropped.

Fast convergence is important in networks that require high availability.

## Static vs Dynamic Routing

| Factor | Static routing | Dynamic routing |
| --- | --- | --- |
| Configuration | manual | automatic through protocols |
| Adaptation | does not adapt automatically | adapts to changes |
| Scalability | poor for large changing networks | better for large networks |
| Overhead | low | higher due to updates |
| Predictability | high | depends on protocol and network state |
| Best for | small stable networks | large or changing networks |

## Local Area Networks

In a LAN, static routes may be enough if the network is small and simple.

Dynamic routing becomes more useful when there are multiple routers, changing paths, redundant links, or frequent topology changes.

## Exam Pattern

When comparing static and dynamic routing, mention configuration, maintenance, scalability, overhead, adaptability, convergence, security, and suitability for the network size and stability.
