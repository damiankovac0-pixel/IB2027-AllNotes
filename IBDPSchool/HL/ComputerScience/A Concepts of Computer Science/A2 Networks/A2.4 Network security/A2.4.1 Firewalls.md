# A2.4.1 Firewalls

A firewall is hardware or software that monitors and filters incoming and outgoing network traffic using security rules.

Firewalls are usually placed between a trusted internal network and an untrusted external network such as the internet.

## What Firewalls Inspect

A basic firewall can inspect packet header information such as:

- source IP address
- destination IP address
- source port
- destination port
- protocol, such as TCP or UDP

Modern next-generation firewalls can also inspect packet contents and application-layer behaviour. This is often called deep packet inspection.

## Firewall Rules

Firewall rules decide whether traffic is allowed or blocked.

A rule may specify:

- source address
- destination address
- port number
- protocol
- direction of traffic
- action, such as allow or deny

Rules are usually processed in order. The first matching rule is applied.

## Allow Lists And Deny Lists

An allow list contains approved traffic sources, destinations, applications, or services.

A deny list contains blocked traffic sources, destinations, applications, or services.

Allow-listing is stricter because anything not explicitly allowed is blocked.

Deny-listing is easier to manage in some cases, but it only blocks known bad traffic.

## What Firewalls Are Good At

Firewalls are effective for:

- controlling access to network resources
- blocking unwanted inbound traffic
- limiting outbound traffic to approved services
- filtering by IP address, domain, protocol, and port
- logging suspicious network activity
- supporting VPN connections in many enterprise setups
- reducing exposure to known malicious sources

## Traffic Monitoring And Logging

Firewalls can log allowed and denied traffic.

Logs help administrators:

- detect suspicious activity
- investigate attempted attacks
- identify misconfigured devices
- check whether rules are working
- provide evidence during incident response

## Application-Level Firewalls

An application-level firewall can inspect traffic at the application layer.

This can help detect traffic that looks normal at the IP or port level but is suspicious at the application level.

Example: traffic on port 443 is usually HTTPS, but the firewall may still inspect whether the application behaviour is suspicious.

## Limitations Of Firewalls

Firewalls are not complete protection.

Limitations include:

- internal threats from malicious or careless users
- attacks that use allowed ports or allowed applications
- encrypted malicious traffic that cannot be inspected easily
- zero-day attacks unknown to security tools
- phishing attacks that trick users directly
- malware already inside the network
- damage caused by poor firewall configuration
- possible latency from traffic inspection

## Firewall Misconfiguration

A misconfigured firewall can create security and availability problems.

Examples:

- allowing too much inbound traffic
- blocking legitimate business traffic
- leaving unused ports open
- processing rules in the wrong order
- failing to log important events

## NAT And Firewalls

[[A2.3.1 IP addressing#Network Address Translation|NAT]] hides internal private IP addresses behind a public IP address.

NAT can make internal devices harder to target directly, but NAT is not a firewall by itself. It should be combined with firewall rules and other controls.

## Evaluation Summary

Firewalls are effective for controlling traffic at network boundaries, enforcing access rules, and logging suspicious activity.

They are less effective against threats that originate inside the network, social engineering, misconfiguration, unknown vulnerabilities, or malicious traffic that looks legitimate.

## Exam Pattern

For a firewall evaluation question, do not just say "it blocks hackers". Discuss strengths, limitations, configuration, logging, internal threats, and why firewalls must be part of a layered security approach.
