# A2.4.2 Network Vulnerabilities

This is an HL syllabus point.

A network vulnerability is a weakness that could be exploited to compromise confidentiality, integrity, availability, or authentication.

## Security Goals

Common security goals are:

- confidentiality: only authorized users can access data
- integrity: data is not altered without authorization
- availability: systems remain usable when needed
- authentication: users and systems prove who they are
- non-repudiation: a party cannot reasonably deny an action they performed

## Distributed Denial Of Service

A distributed denial of service attack, or DDoS attack, uses many devices to overwhelm a target with traffic or requests.

Effect:

- service becomes slow or unavailable
- legitimate users cannot access the system
- business operations may be disrupted

Example: thousands of compromised computers send requests to an e-commerce website until it cannot respond to real customers.

## Insecure Network Protocols

Insecure protocols lack strong encryption or authentication.

Examples:

- HTTP instead of HTTPS
- FTP instead of secure file transfer methods
- Telnet instead of SSH

Risk:

- credentials may be intercepted
- data may be read or altered
- attackers may hijack sessions

## Malware

Malware is malicious software designed to damage, disrupt, spy on, or gain unauthorized access to systems.

Examples:

- viruses
- worms
- ransomware
- spyware
- trojans

Network impact:

- spreads between devices
- steals data
- encrypts files for ransom
- creates botnets for DDoS attacks
- opens backdoors for attackers

## Man-In-The-Middle Attacks

A man-in-the-middle attack, or MitM attack, occurs when an attacker secretly intercepts communication between two parties.

The attacker may:

- read messages
- alter messages
- steal credentials
- impersonate one side

Example: an attacker on public Wi-Fi intercepts payment information sent over an insecure connection.

MitM attacks threaten confidentiality and integrity.

## Phishing

Phishing is a social engineering attack that tricks users into giving sensitive information or taking harmful actions.

Common phishing methods:

- fake bank emails
- fake login pages
- malicious invoice attachments
- urgent account warning messages
- messages pretending to be from trusted staff

Phishing can lead to:

- stolen credentials
- unauthorized access
- identity theft
- malware infection
- financial loss

## SQL Injection

SQL injection happens when an attacker inserts malicious SQL into user input so that the database executes unintended commands.

Possible outcomes:

- revealing private data
- bypassing login checks
- modifying or deleting data
- taking control of database behaviour

Example: an attacker enters crafted SQL into a search box or login field to force the database to return user records.

## Cross-Site Scripting

Cross-site scripting, or XSS, is a web vulnerability where attackers inject malicious scripts into content viewed by other users.

Possible outcomes:

- stealing cookies
- hijacking sessions
- redirecting users
- changing page content
- performing actions as another user

Example: a malicious script is added to a comment field and runs when other users view the comment.

## Unpatched Software

Unpatched software has not received security updates that fix known vulnerabilities.

Risk:

- attackers can use publicly known exploits
- old systems become easy targets
- malware can spread through known weaknesses

Example: a server runs an outdated operating system with a known remote access vulnerability.

## Weak Authentication

Weak authentication means access controls are easy to bypass.

Examples:

- short passwords
- reused passwords
- default passwords
- no multifactor authentication
- poor account recovery checks

Risk:

- brute-force attacks
- credential stuffing
- unauthorized account access

## Zero-Day Exploits

A zero-day exploit uses a vulnerability that is unknown to the software developer or has no available patch yet.

Zero-days are dangerous because normal patching cannot protect against them immediately.

Defence relies on layered protection, monitoring, rapid response, and reducing the damage an attacker can cause.

## Vulnerability Summary

| Vulnerability | Main risk | Security goal affected |
| --- | --- | --- |
| DDoS | service unavailable | availability |
| Insecure protocols | interception or alteration | confidentiality, integrity |
| Malware | damage, theft, disruption | all goals |
| MitM | interception and alteration | confidentiality, integrity |
| Phishing | credential theft or malware | authentication, confidentiality |
| SQL injection | database compromise | confidentiality, integrity |
| XSS | malicious scripts in user browser | confidentiality, integrity |
| Unpatched software | known exploits | all goals |
| Weak authentication | account takeover | authentication |
| Zero-day exploit | unknown weakness exploited | all goals |

## Exam Pattern

For vulnerability questions, define the attack, describe how it works, give a scenario, and identify what harm it causes.
