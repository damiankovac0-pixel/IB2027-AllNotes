# A2.4.3 Network Countermeasures

This is an HL syllabus point.

A network countermeasure is a control used to reduce the chance or impact of a network attack.

Good security uses multiple countermeasures together. A single countermeasure rarely solves the whole problem.

## Matching Threats To Countermeasures

| Threat | Suitable countermeasures |
| --- | --- |
| DDoS | DDoS mitigation tools, rate limiting, traffic filtering, scalable infrastructure |
| Insecure protocols | encrypted protocols, HTTPS, SSH, TLS, VPNs |
| Malware | updates, email filtering, endpoint protection, training, backups |
| MitM | HTTPS, TLS certificates, VPNs, certificate validation |
| Phishing | email filtering, MFA, employee training |
| SQL injection | input validation, parameterized queries, secure coding |
| XSS | content security policy, input validation, output encoding |
| Unpatched software | regular patching and vulnerability management |
| Weak authentication | password policies, MFA, account lockout controls |
| Zero-day exploits | IDS/IPS, segmentation, least privilege, monitoring, rapid response |

## Content Security Policies

A content security policy, or CSP, is a browser-side control that restricts which sources a page can load content from.

It can limit:

- scripts
- stylesheets
- images
- frames
- fonts
- other web resources

CSP helps reduce XSS risk because injected scripts from unapproved sources can be blocked.

## Complex Password Policies

Password policies require users to create stronger passwords.

Rules may include:

- minimum length
- mixed uppercase and lowercase letters
- numbers and symbols
- avoiding recently used passwords
- blocking common passwords

Password policies reduce risk from guessing and brute-force attacks, but they are stronger when combined with MFA.

## Encrypted Protocols

Encrypted protocols protect data in transit.

Examples:

- HTTPS instead of HTTP
- SSH instead of Telnet
- secure file transfer instead of plain FTP

Encryption reduces the risk of eavesdropping and MitM attacks.

## SSL And TLS Certificates

SSL/TLS certificates support HTTPS by proving that a public key belongs to a particular website or organization.

They help users and browsers verify that they are connecting to the intended server rather than an impostor.

TLS is the modern protocol. SSL is older, but the phrase SSL certificate is still commonly used.

## Software Updates

Software updates patch known vulnerabilities in operating systems, applications, firmware, and network devices.

Updating reduces the attack surface because attackers often reuse known exploits against outdated systems.

## DDoS Mitigation Tools

DDoS mitigation tools detect and filter high volumes of malicious traffic.

They may use:

- traffic scrubbing
- rate limiting
- blackholing for extreme cases
- content delivery networks
- filtering by source, pattern, or behaviour

Goal: keep the service available to legitimate users during an attack.

## Input Validation

Input validation checks that user input is safe and in the expected format.

Approaches include:

- filtering unwanted characters
- allow-listing accepted patterns
- rejecting invalid input
- checking type, length, range, and format

Input validation helps reduce SQL injection and XSS risk, but secure coding practices are also needed.

## Email Filtering

Email filtering detects and blocks suspicious email.

It can reduce:

- phishing
- spam
- malicious attachments
- links to known malicious domains

It is not perfect because attackers can craft convincing messages or use compromised legitimate accounts.

## Intrusion Detection Systems

An intrusion detection system, or IDS, monitors network or host activity and alerts administrators about suspicious behaviour.

IDS is passive: it detects and reports, but does not directly block traffic.

Useful for:

- suspicious scans
- unusual login attempts
- policy violations
- possible malware activity

## Intrusion Prevention Systems

An intrusion prevention system, or IPS, monitors activity and can actively block detected attacks.

IPS is active: it can block traffic, reset connections, or deny access.

Risk: a poorly tuned IPS may block legitimate traffic.

## IDS vs IPS

| Feature | IDS | IPS |
| --- | --- | --- |
| Main role | detect and alert | detect and block |
| Traffic effect | passive | active |
| Risk | alerts may be ignored | false positives may block users |
| Best use | monitoring and investigation | prevention and automated response |

## Multi-Factor Authentication

Multi-factor authentication, or MFA, requires more than one proof of identity.

Factors include:

- something you know, such as a password
- something you have, such as an authenticator app or token
- something you are, such as a fingerprint

MFA reduces the damage from stolen passwords because the password alone is not enough.

## Virtual Private Network

A virtual private network, or VPN, creates an encrypted tunnel over an untrusted network.

VPNs help protect data in transit and allow secure remote access to private network resources.

VPNs do not automatically make a device safe. If the user's device is infected, the VPN may still carry malicious traffic into the network.

## Security Testing

Security testing searches for weaknesses before attackers exploit them.

Examples:

- vulnerability scanning
- penetration testing
- configuration audits
- phishing simulations
- firewall rule reviews

Testing also checks whether countermeasures are correctly configured.

## Employee Training

Employees are often targeted through social engineering.

Training helps users:

- recognize phishing
- report suspicious activity
- handle sensitive data correctly
- use passwords and MFA properly
- avoid unsafe downloads and links

Training should be repeated because threats and staff behaviour change over time.

## Wireless Security Measures

Wireless security measures include:

- WPA2 or WPA3 encryption
- strong Wi-Fi passwords
- MAC filtering
- allow lists and deny lists
- disabling unnecessary guest access

MAC filtering can help, but MAC addresses can be spoofed. It should not replace encryption.

## Backups

Backups reduce the impact of ransomware, accidental deletion, or hardware failure.

Good backups should be:

- regular
- tested
- protected from attackers
- stored separately from the main network when possible
- encrypted if they contain sensitive data

## Exam Pattern

The best answers match a specific countermeasure to a specific threat. Explain why it helps and mention one limitation.
