# A3.1 Database Fundamentals

This topic explains the basic structure of relational databases and why they are used instead of flat-file databases.

The syllabus has one main point: explain the features, benefits, and limitations of a relational database.

## Core Idea

A relational database stores data in related tables. Each table represents an entity, each row represents one record, and each column represents an attribute.

Relational databases reduce repeated data and make it easier to query, update, protect, and maintain data.

## Essential Vocabulary

- database: an organized collection of data
- entity: a table representing something data is stored about
- attribute: a characteristic or field of an entity
- record: one row in a table
- tuple: another word for one row or record
- primary key: a unique identifier for each record
- foreign key: a field that refers to a primary key in another table
- relationship: a link between entities
- cardinality: the type of relationship between entities
- modality: whether a relationship is required or optional
- normalization: organizing data to reduce duplication and anomalies
