# A3.1.1 Relational Database Features

A database is an organized collection of structured data.

A relational database stores data in related tables.

## Flat-File Database

A flat-file database stores data in a single table.

This can work for very small simple datasets, but it becomes inefficient when data is repeated.

Example: a veterinary practice stores owner, patient, vet, and treatment data in one large table. The same owner address and vet details may appear many times.

## Problems With Flat-File Databases

Flat-file databases can cause:

- repeated data
- wasted storage
- update anomalies
- insertion anomalies
- deletion anomalies
- inconsistent data
- poor scalability

## Anomalies

An anomaly is a data problem caused by poor database structure.

### Update Anomaly

An update anomaly happens when repeated data is changed in one place but not another.

Example: an owner changes address, but only some of their repeated records are updated. The database now contains conflicting addresses.

### Insertion Anomaly

An insertion anomaly happens when data cannot be added unless unrelated data is also added.

Example: a new vet cannot be added until the vet has treated a patient, because the flat file stores vet details only inside treatment records.

### Deletion Anomaly

A deletion anomaly happens when deleting one record accidentally deletes other important information.

Example: deleting the only treatment record for a vet also removes all information about that vet.

## Relational Database Structure

A relational database splits data into separate related tables.

Example entities for a veterinary database:

- OWNER
- PATIENT
- VET
- TREATMENT

Each table stores data about one type of thing.

## Entity

An entity is something data is stored about.

In a relational database, an entity is usually represented as a table.

Examples:

- customer
- product
- order
- student
- course
- patient

## Attribute

An attribute is a characteristic or field of an entity.

Example attributes for a STUDENT entity:

- StudentID
- FirstName
- LastName
- DateOfBirth
- Email

In a table, attributes are columns.

## Record And Tuple

A record is one row in a table.

A tuple is another term for one row in a relational database.

Example: one specific student in the STUDENT table is one record.

## Normalization

Normalization is the process of organizing database tables to reduce duplication and anomalies.

The basic idea is to separate data into logical entities and connect them using keys.

Benefits of normalization:

- less repeated data
- fewer update anomalies
- fewer deletion anomalies
- fewer insertion anomalies
- better data consistency

## Relational Database Feature Summary

| Feature | Meaning |
| --- | --- |
| Table/entity | stores one category of data |
| Row/record/tuple | one instance of an entity |
| Column/attribute | one field or characteristic |
| Primary key | uniquely identifies a row |
| Foreign key | links to a row in another table |
| Relationship | connection between entities |
| Normalization | reduces repetition and anomalies |

## Exam Pattern

If asked why relational databases are used, contrast them with flat files: relational databases split data into entities, reduce duplicated data, use keys to link tables, and reduce anomalies.
