# A3.2 Database Design

Database design is the process of planning tables, fields, keys, relationships, data types, and constraints before building a database.

Good database design reduces duplicated data, prevents anomalies, improves data integrity, and makes querying easier.

## Design Workflow

1. Identify entities and attributes.
2. Draw an ERD with relationships, cardinality, and modality.
3. Choose suitable primary keys and foreign keys.
4. Choose suitable data types.
5. Normalize to reduce duplication and anomalies.
6. Consider denormalization only when performance needs justify it.
