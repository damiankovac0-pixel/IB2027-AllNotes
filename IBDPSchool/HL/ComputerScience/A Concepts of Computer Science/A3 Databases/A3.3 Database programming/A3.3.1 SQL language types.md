# A3.3.1 SQL Language Types

SQL is used to define, manipulate, control, and manage data in relational databases.

The main SQL language types are:

- DDL: data definition language
- DML: data manipulation language
- DCL: data control language
- TCL: transaction control language

## Data Definition Language

Data definition language, or DDL, is used to create, modify, and remove database structures.

DDL affects the schema rather than just individual records.

Common DDL commands:

- CREATE
- ALTER
- DROP
- TRUNCATE
- RENAME

## CREATE

CREATE makes a new database object such as a database, table, view, or index.

Example:

```sql
CREATE TABLE Customer (
  CustomerID INTEGER NOT NULL PRIMARY KEY,
  CustomerEmail VARCHAR(255),
  CustomerAddress VARCHAR(255)
);
```

## ALTER

ALTER changes an existing database structure.

Example:

```sql
ALTER TABLE Customer ADD PhoneNumber VARCHAR(20);
```

## DROP

DROP removes a database object.

Example:

```sql
DROP TABLE Customer;
```

DROP is destructive because the structure is removed.

## Data Manipulation Language

Data manipulation language, or DML, is used to retrieve and modify records stored in tables.

Common DML commands:

- SELECT
- INSERT
- UPDATE
- DELETE

## SELECT

SELECT retrieves data.

Example:

```sql
SELECT CustomerID, CustomerEmail
FROM Customer;
```

## INSERT

INSERT adds new records.

Example:

```sql
INSERT INTO Customer (CustomerID, CustomerEmail)
VALUES (1, 'ana@example.com');
```

## UPDATE

UPDATE modifies existing records.

Example:

```sql
UPDATE Customer
SET CustomerEmail = 'new@example.com'
WHERE CustomerID = 1;
```

## DELETE

DELETE removes records from a table.

Example:

```sql
DELETE FROM Customer
WHERE CustomerID = 1;
```

## Data Control Language

Data control language, or DCL, controls access to the database.

Common DCL commands:

- GRANT
- REVOKE

Example:

```sql
GRANT SELECT ON Customer TO reporting_user;
```

DCL supports security by controlling who can read, insert, update, or delete data.

## Transaction Control Language

Transaction control language, or TCL, manages transactions.

Common TCL commands:

- COMMIT
- ROLLBACK
- SAVEPOINT

Example:

```sql
BEGIN TRANSACTION;
UPDATE Account SET Balance = Balance - 100 WHERE AccountID = 1;
UPDATE Account SET Balance = Balance + 100 WHERE AccountID = 2;
COMMIT;
```

## Language Type Summary

| Type | Purpose | Example commands |
| --- | --- | --- |
| DDL | define database structures | CREATE, ALTER, DROP |
| DML | retrieve and modify data | SELECT, INSERT, UPDATE, DELETE |
| DCL | control access | GRANT, REVOKE |
| TCL | manage transactions | COMMIT, ROLLBACK |

## Exam Pattern

Be able to classify SQL commands by purpose. A common mistake is confusing DELETE and DROP: DELETE removes records, while DROP removes the database object itself.
