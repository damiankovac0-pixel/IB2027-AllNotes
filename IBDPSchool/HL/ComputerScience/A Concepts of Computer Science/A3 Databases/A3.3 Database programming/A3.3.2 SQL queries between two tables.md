# A3.3.2 SQL Queries Between Two Tables

SQL queries retrieve data from tables.

Queries between two tables usually use a JOIN so related records can be combined.

## Basic SELECT

Retrieve specific columns:

```sql
SELECT CustomerID, CustomerEmail
FROM Customer;
```

Retrieve all columns:

```sql
SELECT *
FROM Customer;
```

## WHERE

WHERE filters records using a condition.

```sql
SELECT *
FROM Item
WHERE ItemCost > 300;
```

## Relational Operators

| Operator | Meaning |
| --- | --- |
| = | equal to |
| <> | not equal to |
| > | greater than |
| < | less than |
| >= | greater than or equal to |
| <= | less than or equal to |

Example:

```sql
SELECT *
FROM Employee
WHERE Salary >= 50000;
```

## JOIN

JOIN combines rows from two tables using a related field, usually a primary key and foreign key.

Example:

```sql
SELECT Purchase.Date, Customer.CustomerEmail
FROM Purchase
JOIN Customer ON Purchase.CustomerID = Customer.CustomerID;
```

This connects purchases to customer details.

## DISTINCT

DISTINCT returns unique values and removes duplicates from the result.

```sql
SELECT DISTINCT CustomerCountry
FROM Customer;
```

## BETWEEN

BETWEEN filters values within a range.

```sql
SELECT *
FROM Purchase
WHERE Date BETWEEN '2025-07-28' AND '2025-07-31';
```

## IN

IN checks whether a value matches one value in a list.

```sql
SELECT *
FROM Customer
WHERE CustomerCountry IN ('Italy', 'France', 'Spain');
```

## IS NULL And IS NOT NULL

NULL means missing or unknown data.

```sql
SELECT *
FROM Employee
WHERE ManagerID IS NULL;
```

```sql
SELECT *
FROM Employee
WHERE ManagerID IS NOT NULL;
```

Do not use `= NULL`; use `IS NULL`.

## AND And OR

AND requires both conditions to be true.

OR requires at least one condition to be true.

```sql
SELECT *
FROM Customer
WHERE CustomerCountry = 'Italy'
   OR CustomerCountry = 'France';
```

Use brackets when mixing AND and OR to avoid ambiguous logic.

## LIKE

LIKE performs pattern matching.

Wildcards:

- `%` means any sequence of characters
- `_` means one single character

Example:

```sql
SELECT *
FROM Artist
WHERE ArtistName LIKE '%work%';
```

## ORDER BY

ORDER BY sorts results.

```sql
SELECT *
FROM Item
ORDER BY ItemCost DESC, ItemDesc ASC;
```

`ASC` means ascending.

`DESC` means descending.

## GROUP BY

GROUP BY groups records so aggregate functions can calculate summaries.

```sql
SELECT CustomerCountry, COUNT(CustomerID) AS CustomerCount
FROM Customer
GROUP BY CustomerCountry;
```

## HAVING vs WHERE

WHERE filters rows before grouping.

HAVING filters groups after grouping.

```sql
SELECT CustomerCountry, COUNT(CustomerID) AS CustomerCount
FROM Customer
GROUP BY CustomerCountry
HAVING COUNT(CustomerID) > 10;
```

## Exam Pattern

For two-table SQL questions, identify the shared key first, then write the JOIN condition. Most errors come from missing the ON condition or filtering with the wrong table field.
