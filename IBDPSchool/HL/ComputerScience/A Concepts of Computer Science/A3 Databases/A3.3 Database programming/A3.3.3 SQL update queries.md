# A3.3.3 SQL Update Queries

SQL can update data by adding, modifying, or deleting records.

The main update commands are:

- INSERT
- UPDATE
- DELETE

## INSERT

INSERT adds a new record to a table.

Syntax:

```sql
INSERT INTO table_name (field1, field2, field3)
VALUES (value1, value2, value3);
```

Example:

```sql
INSERT INTO Customer (CustomerID, CustomerEmail, CustomerAddress)
VALUES (2, 'christopher@example.com', '34 Maple Crescent');
```

## UPDATE

UPDATE changes existing records.

Syntax:

```sql
UPDATE table_name
SET field1 = value1, field2 = value2
WHERE condition;
```

Example:

```sql
UPDATE Item
SET ArtistID = 101
WHERE ItemID = 2940;
```

## DELETE

DELETE removes records from a table.

Syntax:

```sql
DELETE FROM table_name
WHERE condition;
```

Example:

```sql
DELETE FROM Item
WHERE ItemID = 49393;
```

## Importance Of WHERE

WHERE is critical in UPDATE and DELETE queries.

Without WHERE, the command may affect every record in the table.

Dangerous example:

```sql
DELETE FROM Customer;
```

This removes all records from Customer.

## Indexes And Update Performance

An index helps the database find records faster.

Indexes improve read performance, especially in large databases.

However, indexes can slow update operations because the index must also be updated when data changes.

Update operations affected by indexes include:

- adding records
- modifying indexed fields
- deleting records

## Why Indexes Can Slow Updates

When an indexed field changes, the database must:

- update the table data
- update the index structure
- possibly reorganize index pages
- manage locks or blocking for concurrent transactions

This extra work can reduce performance.

## Reducing Update Problems

Possible strategies:

- batch large updates
- avoid unnecessary indexes
- rebuild or reorganize indexes when needed
- update during low-traffic periods
- design indexes around actual query needs

## Exam Pattern

Be ready to explain how INSERT, UPDATE, and DELETE modify data. For safety, mention that UPDATE and DELETE should normally include a WHERE condition.
