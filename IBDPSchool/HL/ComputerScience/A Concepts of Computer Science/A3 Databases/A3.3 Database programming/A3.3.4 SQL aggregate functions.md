# A3.3.4 SQL Aggregate Functions

This is an HL syllabus point.

Aggregate functions perform calculations across multiple rows and return summary values.

Common aggregate functions:

- COUNT
- SUM
- AVG
- MIN
- MAX

## COUNT

COUNT returns the number of matching rows or values.

```sql
SELECT COUNT(ItemID)
FROM Purchase;
```

COUNT with a condition:

```sql
SELECT COUNT(ItemID)
FROM Purchase
WHERE Quantity > 1;
```

COUNT DISTINCT counts unique values only.

```sql
SELECT COUNT(DISTINCT ItemID)
FROM Purchase;
```

## SUM

SUM returns the total of a numeric field.

```sql
SELECT SUM(ItemCost)
FROM Item;
```

With a condition:

```sql
SELECT SUM(ItemCost)
FROM Item
WHERE ArtistID = 105;
```

## AVG

AVG returns the mean average of a numeric field.

```sql
SELECT AVG(Quantity)
FROM Purchase
WHERE CustomerID = 3;
```

## MIN And MAX

MIN returns the smallest value.

MAX returns the largest value.

```sql
SELECT MIN(Salary) AS MinSalary, MAX(Salary) AS MaxSalary
FROM Employee;
```

## Aggregates With GROUP BY

GROUP BY is used when aggregate results are needed per group.

Example: number of employees in each department.

```sql
SELECT Department.DepartmentName,
       COUNT(Employee.EmployeeID) AS EmployeeCount
FROM Employee
JOIN Department ON Employee.DepartmentID = Department.DepartmentID
GROUP BY Department.DepartmentName;
```

Example: average salary per department.

```sql
SELECT Department.DepartmentName,
       AVG(Employee.Salary) AS AvgSalary
FROM Employee
JOIN Department ON Employee.DepartmentID = Department.DepartmentID
GROUP BY Department.DepartmentName;
```

## HAVING With Aggregates

HAVING filters grouped results.

```sql
SELECT DepartmentID, COUNT(EmployeeID) AS EmployeeCount
FROM Employee
GROUP BY DepartmentID
HAVING COUNT(EmployeeID) > 5;
```

## Common Mistakes

- using WHERE to filter aggregate groups instead of HAVING
- forgetting GROUP BY when selecting non-aggregated columns
- using aggregate functions on text fields where numeric fields are required
- confusing COUNT(*) with COUNT(field), since COUNT(field) may ignore NULL values

## Exam Pattern

For aggregate questions, identify the grouping field and the calculation field. Use GROUP BY when the question asks for a result per category.
