# A3.4.1 Types Of Databases

Different database models store and organize data in different ways.

The best model depends on the data, scale, performance needs, and application purpose.

## Relational Databases

Relational databases store data in tables with relationships between them.

Best for:

- structured data
- transactions
- consistency
- SQL querying
- systems with clear entities and relationships

Examples:

- banking systems
- school information systems
- booking systems
- inventory systems

## NoSQL Databases

NoSQL databases are designed for large volumes of data, flexible schemas, and diverse data types.

They do not store data primarily as related tables.

Common NoSQL models:

- document databases
- key-value stores
- column-family databases
- graph databases

## Document Databases

Document databases store data as documents, often in JSON-like structures.

Example: a book document may store title, author, publisher, genres, and reviews together instead of splitting them into many related tables.

Advantages:

- flexible schema
- useful for varied data
- easier to store nested structures
- scales well for many web applications

Limitations:

- weaker fit for complex transactions than relational databases
- data may be duplicated
- consistency can be harder to manage

## Cloud Databases

A cloud database runs on a cloud computing platform.

Benefits:

- scalability
- high availability
- flexible resource management
- pay-as-you-use pricing
- easier global access

Cloud databases can be relational or NoSQL.

## Self-Managed Cloud Databases

In a self-managed cloud database, the organization runs the database on cloud infrastructure.

The cloud provider manages infrastructure such as servers and storage, while the organization manages the database setup, updates, backups, and administration.

Advantage: more control.

Disadvantage: more maintenance responsibility.

## Managed Database Services

In a managed database service, the vendor manages much of the database infrastructure and administration.

Advantages:

- less administrative work
- automated backups and updates may be available
- easier scaling

Disadvantages:

- less control
- possible vendor lock-in
- ongoing service costs

## Spatial Databases

Spatial databases store and query data related to location and geometry.

They are optimized for data such as:

- points
- lines
- polygons
- coordinates
- maps

Uses:

- navigation systems
- geographic information systems
- delivery routing
- urban planning
- geospatial analysis

## In-Memory Databases

In-memory databases store data mainly in RAM rather than on disk.

Advantages:

- very fast access
- low latency
- useful for real-time processing

Uses:

- real-time banking
- medical devices
- online gaming
- sensor data processing
- e-commerce recommendation systems
- machine learning pipelines

Limitations:

- RAM is more expensive than disk storage
- persistence and recovery must be carefully designed
- not ideal for every large long-term storage problem

## Summary Table

| Type | Main strength | Typical use |
| --- | --- | --- |
| Relational | structured data and transactions | banking, booking, school systems |
| NoSQL | flexible schema and scale | web apps, big data, varied records |
| Cloud | scalable managed infrastructure | distributed online services |
| Spatial | location and geometry queries | maps, routing, GIS |
| In-memory | speed and low latency | real-time systems |

## Exam Pattern

For database type questions, match the database model to the scenario. Explain why the structure, scale, performance, or flexibility fits the problem.
