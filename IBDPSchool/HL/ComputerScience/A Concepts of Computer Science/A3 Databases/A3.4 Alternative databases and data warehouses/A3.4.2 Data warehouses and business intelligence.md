# A3.4.2 Data Warehouses And Business Intelligence

A data warehouse is a large store of historical data used for analysis, reporting, and business intelligence.

Business intelligence uses technologies, applications, and practices to collect, integrate, analyse, and present business information for decision-making.

## Purpose Of Data Warehouses

Data warehouses help organizations:

- integrate data from many sources
- analyse historical trends
- support reports and dashboards
- improve decision-making
- identify customer behaviour patterns
- compare business performance over time

## Transactional Databases vs Data Warehouses

| Transactional database | Data warehouse |
| --- | --- |
| supports day-to-day operations | supports analysis and reporting |
| current operational data | historical data |
| frequent inserts and updates | mostly loaded and queried |
| optimized for transactions | optimized for analysis |
| used by operational systems | used by analysts and managers |

## Main Characteristics

Data warehouses are commonly described as:

- subject-oriented
- integrated
- time-variant
- non-volatile

## Subject-Oriented

A data warehouse is organized around key business subjects.

Examples:

- sales
- customers
- marketing
- finance
- inventory

This helps users analyse one area of the business rather than individual transactions only.

## Integrated

A data warehouse gathers data from multiple sources and combines it into a consistent format.

Sources may include:

- operational databases
- flat files
- logs
- APIs
- external data sources

Integration requires cleaning and transforming data so it can be analysed together.

## Time-Variant

Data warehouses store historical data.

This allows analysis of changes over time.

Examples:

- sales by month
- customer spending over years
- product demand across seasons
- regional growth trends

## Non-Volatile

Data in a warehouse is generally not deleted or constantly changed like operational data.

Once loaded, it is mainly used for analysis.

This supports reliable historical comparison.

## Append-Only

Append-only means new versions of data are added rather than replacing old records.

This allows analysts to see how data changed over time.

## ETL Process

ETL stands for extract, transform, and load.

## Extract

Extract means gathering data from different sources.

Examples:

- sales database
- customer database
- website logs
- loyalty card system
- supplier API

## Transform

Transform means preparing data for analysis.

This may include:

- cleaning invalid data
- removing duplicates
- standardizing formats
- combining fields
- filtering irrelevant records
- converting data types

## Load

Load means placing the transformed data into the data warehouse.

## Data Cleaning

Data cleaning improves quality before analysis.

Examples:

- correcting inconsistent date formats
- removing duplicate customer records
- handling missing values
- standardizing country names
- validating numeric ranges

## Exam Pattern

Explain that data warehouses are not mainly for everyday transactions. They integrate historical data from multiple sources to support analysis, reporting, and business intelligence.
