# A3.4.4 Distributed Databases

A distributed database stores data across two or more sites, servers, or networks while presenting a unified view to users and applications.

The data may be located on the same network or across different geographical locations.

## Why Distributed Databases Are Used

Distributed databases are used to support:

- large scale systems
- geographical distribution
- high availability
- fault tolerance
- local access to nearby data
- high transaction volumes

Examples:

- online retailers managing inventory across warehouses
- telecom companies managing subscriptions and network traffic
- banks operating across regions
- global applications with users in many countries

## Unified View

Although data is stored in multiple locations, users should be able to interact with it as one logical database.

This hides the complexity of the physical distribution.

## Data Consistency

Data consistency means all copies of data reflect the correct state.

In distributed systems, consistency is difficult because updates may occur at different locations.

Strong consistency means updates are visible to all nodes immediately or as if they happened in a single agreed order.

Weak or eventual consistency allows temporary differences, but nodes should become consistent later.

## Concurrency Control

Concurrency control manages simultaneous access to shared data.

Without it, two users or systems might update the same data at the same time and create incorrect results.

One solution is locking:

1. A transaction locks the data it needs.
2. Other transactions wait or are blocked.
3. The transaction commits or rolls back.
4. The lock is released.

This protects integrity but can reduce performance.

## Data Partitioning

Data partitioning splits data across multiple nodes or servers.

Benefits:

- improved scalability
- better performance
- easier management of large datasets
- local access to relevant data
- ability to handle high transaction rates

Example: customers in Europe may be stored on European servers, while customers in Asia may be stored on Asian servers.

## Replication

Replication stores copies of data on multiple nodes.

Benefits:

- improved availability
- fault tolerance
- faster local reads

Challenges:

- keeping copies synchronized
- handling conflicting updates
- increased storage requirements

## Data Security

Distributed databases must protect data across multiple locations.

Security measures include:

- encryption for stored data
- encryption for data in transit
- role-based access control
- strong authentication
- monitoring and auditing
- secure backups
- network security controls

Security is harder because there are more systems, connections, users, and locations to protect.

## Benefits

- scalable across many servers
- improved availability
- better performance for geographically spread users
- local control of local data
- fault tolerance if one site fails

## Limitations

- complex consistency management
- difficult concurrency control
- more complex security
- higher administration complexity
- network latency can affect performance
- failures can be harder to diagnose

## Exam Pattern

For distributed database questions, mention multiple sites, unified access, consistency, partitioning, security, and concurrency control. Strong answers explain both the scalability benefits and the management complexity.
