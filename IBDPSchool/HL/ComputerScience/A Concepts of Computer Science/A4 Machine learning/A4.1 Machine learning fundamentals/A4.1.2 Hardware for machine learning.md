# A4.1.2 Hardware For Machine Learning

Machine learning hardware depends on what stage the model is in: development, data preparation, training, deployment, or edge use.

The main factors are:

- data set size
- model complexity
- required speed
- whether real-time response is needed
- cost
- memory and storage needs
- power consumption
- whether the system must scale

## Development And Testing

This is the early stage where code is written and models are tested with smaller data sets.

Typical hardware:

- standard laptop or desktop
- multi-core CPU
- 8-16 GB RAM for basic work
- SSD storage
- optional low-end GPU

Good for learning, experimenting, and small models. Not suitable for training large commercial deep learning systems.

## Data Processing And Feature Engineering

This stage prepares the data before training. It connects directly to [[A4.2 Data preprocessing]].

Typical hardware:

- strong CPU
- large RAM
- fast SSDs
- servers or workstations for larger data sets

The focus is usually data cleaning, transformation, and feature preparation rather than raw neural-network training power.

## Model Training And Deep Learning

Training complex models, especially [[A4.1.1 Types of machine learning#Deep Learning|deep learning]] models, needs much more processing power.

Typical hardware:

- dedicated GPU servers
- multiple high-end GPUs
- TPUs for suitable tensor-heavy workloads
- large RAM
- high-speed storage

The reason is that neural networks involve large numbers of matrix and vector calculations.

## Large-Scale Deployment And Production

Deployment means using the trained model in a real system.

Production systems need:

- reliability
- low response time
- scalability
- strong networking
- monitoring
- storage and backup systems

Typical hardware:

- high-end servers
- cloud platforms
- scalable CPUs, GPUs, or TPUs
- enterprise or distributed storage

## Edge Computing

Edge computing means processing data close to where it is generated instead of sending everything to a distant cloud server.

Examples:

- smartphone facial recognition
- autonomous vehicle decisions
- smart cameras
- IoT sensors
- medical monitoring devices

Advantages:

- lower latency
- less network traffic
- can improve privacy because less data leaves the device
- useful when internet access is unreliable

Limitations:

- less processing power than cloud systems
- limited storage and memory
- power consumption matters
- physical hardware must be maintained

## CPU

A CPU is a general-purpose processor.

Good for:

- normal computing
- development and testing
- data processing
- small ML tasks

Less good for large deep learning tasks because it is not optimized for massive parallel processing.

## GPU

A GPU is designed for parallel processing. It has many smaller cores that can perform similar calculations at the same time.

This is useful for machine learning because neural networks use many matrix and vector operations.

Good for:

- deep learning training
- image processing
- large data sets
- speeding up experimentation

## NPU

An NPU is a neural processing unit designed to accelerate neural-network calculations, often in consumer devices.

Good for:

- on-device AI
- low-power inference
- laptops and phones with AI features

## TPU

A TPU is a tensor processing unit designed by Google for tensor calculations, especially large neural-network workloads.

Good for:

- TensorFlow workloads
- large-scale deep learning
- cloud-based training
- efficient tensor and matrix operations

## ASIC

An ASIC is a custom chip designed for a specific task.

Strengths:

- very efficient
- low power use
- high performance for one fixed purpose

Limitations:

- expensive to design
- not flexible
- not reprogrammable

Best when the task is stable and deployed at large scale.

## FPGA

An FPGA is hardware that can be reprogrammed after manufacturing.

Strengths:

- more flexible than an ASIC
- useful for prototyping
- can accelerate specific workloads

Limitations:

- requires specialist knowledge
- usually less efficient than a custom ASIC for a fixed task

## Cloud-Based Platforms

Cloud platforms provide computing resources over the internet.

Examples:

- AWS
- Google Cloud
- Microsoft Azure
- Google Colab

Advantages:

- scalable CPUs, GPUs, TPUs, storage, and networking
- no need to buy all hardware yourself
- useful for large models
- can deploy models as APIs

Limitations:

- subscription costs
- dependence on an external provider
- privacy and security concerns when data is sent off-site

## High-Performance Computing Centres

High-performance computing centres, or HPC centres, are large facilities built for extremely demanding computation.

They are often used for scientific or academic research and may require institutional access, grants, or allocated compute time.

Good for:

- huge data sets
- complex simulations
- very large model training
- research workloads

## Hardware Choice Summary

| Scenario | Suitable hardware | Reason |
| --- | --- | --- |
| Small student project | laptop or desktop | enough for simple models |
| Data preprocessing | strong CPU, RAM, SSD | data cleaning and transformation need memory and storage speed |
| Deep learning training | GPU or TPU | parallel tensor/matrix operations |
| Large deployment | cloud or high-end servers | scalability and reliability |
| Real-time local decisions | edge device with NPU, ASIC, FPGA, or small GPU | low latency and low power |
| Fixed task at huge scale | ASIC | efficient but not flexible |
| Task may change | FPGA | reprogrammable |
