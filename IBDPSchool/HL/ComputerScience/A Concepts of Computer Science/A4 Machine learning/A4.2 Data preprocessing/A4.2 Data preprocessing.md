# A4.2 Data Preprocessing

Data preprocessing is the stage where raw data is prepared before it is used to train a [[A4.1.0 AI and ML foundations#Machine Learning|machine learning]] model.

The main idea is simple: a model can only learn from the data it is given. If the data is messy, incomplete, biased, duplicated, badly formatted, or irrelevant, the model may learn those problems too.

> Good data does not guarantee a good model, but poor data almost guarantees a poor one.

## Where It Fits In The ML Workflow

A typical machine learning workflow is roughly:

```text
define problem -> gather data -> preprocess data -> explore data -> select features -> split data -> choose model -> train -> evaluate -> tune -> deploy -> monitor
```

Preprocessing happens before training because the model should not be trained on raw, unreliable data.

It includes things like:

- fixing incorrect values
- removing or handling missing values
- removing duplicates
- standardizing formats
- scaling values with [[A4.2.1 Data cleaning#Normalization|normalization]] or [[A4.2.1 Data cleaning#Standardization|standardization]]
- choosing useful [[A4.2.2 Feature selection|features]]
- reducing unnecessary [[A4.2.3 Dimensionality reduction|dimensions]]

## Why It Matters

The quality of preprocessing affects:

- **accuracy**: cleaner data helps the model make better predictions
- **generalization**: the model is more likely to work on new unseen data
- **fairness**: biased data can make biased predictions
- **speed**: fewer irrelevant features means faster training
- **interpretability**: simpler data often makes the model easier to understand

The classic computing phrase is **garbage in, garbage out**. If the input data is flawed, the output will probably be flawed too.

## Quick Example

Imagine a company trains a recruitment model using old CV data. If most successful historical applicants were men, the model may learn that male candidates are more suitable, even if gender is not directly used as a feature.

The issue is not only technical. It is also ethical. Data preprocessing must check whether the data is representative, fair, and suitable for the decision being made.

## Main Distinction

Data preprocessing is not the same as choosing the final machine learning model.

Preprocessing asks:

> Is the data ready and suitable for learning?

Model selection asks:

> Which algorithm should learn from this data?

Both matter, but bad preprocessing can ruin even a good algorithm.
