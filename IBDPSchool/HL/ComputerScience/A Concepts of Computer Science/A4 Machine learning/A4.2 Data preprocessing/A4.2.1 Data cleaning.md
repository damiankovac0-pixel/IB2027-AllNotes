# A4.2.1 Data Cleaning

Data cleaning is the process of making a data set accurate, consistent, complete, and usable before it is given to a [[A4.1.0 AI and ML foundations#Machine Learning Model|machine learning model]].

Raw data is often noisy. In this topic, **noise** means unnecessary, incorrect, inconsistent, or misleading data.

## Why Data Cleaning Matters

A model learns patterns from its training data. If the training data contains errors, the model can learn those errors as if they were real patterns.

Clean data improves:

- prediction accuracy
- reliability
- generalization to unseen data
- fairness and bias control
- legal and regulatory compliance

Poor data can cause:

- unreliable predictions
- [[A4.2.3 Dimensionality reduction#Overfitting|overfitting]] to noise
- biased outputs
- misleading evaluation results

## Common Data Problems

### Outliers

An **outlier** is a value that is very different from the normal pattern in the data.

Examples:

- a person recorded as being 250 years old
- a negative salary
- a house price entered with one extra zero

Outliers can be real unusual cases or data-entry errors. The point is not to delete them automatically; the point is to decide whether they are meaningful or harmful.

Ways to handle outliers:

- **trim** them by removing them
- **cap** them by replacing extreme values with the nearest acceptable value
- **transform** them using methods such as log or square-root transformation to reduce their effect

Outliers can be detected using methods such as IQR, Z-scores, or box plots.

### Duplicate Data

Duplicate data means the same record appears more than once. Duplicates can make a model treat repeated cases as more important than they really are.

Ways to handle duplicates:

- remove exact duplicate records
- use keys or combinations of fields to identify duplicates
- manually review near-duplicates
- consolidate repeated records into one record, for example by averaging values or choosing the most frequent category

This matters because duplicates can skew a model toward over-represented values.

### Incorrect Data

Incorrect data contains wrong values.

Examples:

- negative income where only positive income is possible
- impossible dates
- invalid email addresses
- spelling differences that create separate categories by mistake

Ways to handle incorrect data:

- apply validation rules, such as allowed ranges or formats
- cross-check against another data source
- manually correct small data sets
- use anomaly detection to flag suspicious records

### Irrelevant Data

Irrelevant data does not help solve the problem being modelled.

Example: if a model predicts house prices, the colour of the current owner's car is probably irrelevant.

Irrelevant data can:

- slow training
- make the model harder to interpret
- increase the chance of learning noise
- reduce performance on new data

Filtering irrelevant data connects directly to [[A4.2.2 Feature selection|feature selection]].

### Improperly Formatted Data

Improperly formatted data does not follow the format expected by the model or analysis tool.

Examples:

- dates written as `01-02-2020`, `12/15/2019`, and `2019-10-20` in the same column
- currency values mixed between euros and dollars
- images with different sizes or orientations
- numbers stored as text, such as `thirty` instead of `30`

Ways to fix formatting:

- standardize dates into one format, such as `YYYY-MM-DD`
- convert all units and currencies into a common format
- parse full names into first name and last name if needed
- resize or rotate images consistently
- use regular expressions to identify patterns in text

### Missing Data

Missing data means expected values are absent.

There is no single best fix. The correct choice depends on the problem and how much data is missing.

Common approaches:

- **imputation**: replace missing values with the mean, median, or mode
- **K-NN imputation**: fill missing values using similar records nearby in feature space
- **deletion**: remove records with missing values
- **predictive modelling**: use another model, such as regression or a decision tree, to estimate the missing value

Be careful: filling in missing data can improve completeness, but it can also introduce bias if the replacement values do not match reality.

## Normalization

Normalization, also called min-max scaling, rescales values to a fixed range, usually `0 to 1` or `-1 to 1`.

It is useful when features use very different scales.

Example:

- age might range from 0 to 100
- income might range from 0 to 100000

Without scaling, the model may treat income as more important just because its numbers are larger.

Normalization is useful for algorithms sensitive to scale, including many gradient-based and distance-based methods.

Main weakness: normalization can be strongly affected by outliers, because one extreme value can compress most other values into a small range.

## Standardization

Standardization, also called Z-score normalization, rescales values so the feature has:

- mean of `0`
- standard deviation of `1`

Unlike normalization, standardization does not force values into a fixed range.

It is useful when:

- the minimum and maximum values are unknown
- there are outliers
- the algorithm assumes data is roughly normally distributed

## Normalization vs Standardization

| Method | What it does | Best when |
| --- | --- | --- |
| [[#Normalization|Normalization]] | Scales values into a fixed range such as `0 to 1` | Features have known min/max values and few extreme outliers |
| [[#Standardization|Standardization]] | Scales values around mean `0` and standard deviation `1` | Features have unknown ranges or outliers matter |

Both methods aim to stop scale differences from misleading the model.

## Exam Pattern

If asked why data cleaning is significant, say that it improves the quality of the training data, which improves model accuracy, reliability, fairness, and generalization. Then give examples such as handling missing values, removing duplicates, correcting invalid values, and standardizing formats.
