# A4.2.2 Feature Selection

Feature selection is the process of choosing the most useful input variables for a [[A4.1.0 AI and ML foundations#Machine Learning Model|machine learning model]] and removing features that are irrelevant, redundant, or harmful.

A **feature** is an input attribute used by the model.

Examples in a house-price model:

- floor area
- number of bedrooms
- location
- age of property
- distance to schools
- crime rate

The goal is not to delete data randomly. The goal is to keep the features that help prediction and remove the ones that make learning harder.

## Why Feature Selection Matters

Good feature selection can:

- improve model accuracy
- reduce [[A4.2.3 Dimensionality reduction#Overfitting|overfitting]]
- make training faster
- make the model easier to understand
- remove noise
- reduce unnecessary complexity

Sometimes feature quality matters more than the choice of algorithm. A simple model with useful features can outperform a complex model with messy or irrelevant features.

## Redundant vs Irrelevant Features

### Redundant Features

A redundant feature repeats information already provided by another feature.

Example: in a house data set, `floor area in m2` and `floor area in ft2` describe the same thing. Keeping both may not add useful information.

Highly correlated features can also be redundant. If two features move together almost perfectly, one may be removed with little loss.

### Irrelevant Features

An irrelevant feature has little or no useful relationship to the target output.

Example: if predicting house prices, the seller's favourite colour probably has no meaningful link to sale price.

Irrelevant features can make the model learn noise instead of real patterns.

## Feature Selection Strategies

There are three main strategies in the syllabus: [[#Filter Methods|filter methods]], [[#Wrapper Methods|wrapper methods]], and [[#Embedded Methods|embedded methods]].

## Filter Methods

Filter methods use statistical tests or scores to rank features before the model is trained.

Examples of statistics used:

- correlation coefficients
- Chi-square tests
- ANOVA
- mutual information scores

For a house-price model, a filter method might show that floor area and number of bedrooms are strongly related to price, while a weakly related feature could be removed.

Advantages:

- fast
- less computationally expensive
- independent of the model type
- useful for reducing the feature space early

Limitations:

- may miss interactions between features
- a feature that looks weak alone might be useful when combined with another feature

## Wrapper Methods

Wrapper methods test different subsets of features by actually training and evaluating a model.

Common examples:

- forward selection: start with no features and add useful ones
- backward elimination: start with all features and remove weak ones
- recursive feature elimination, or RFE: repeatedly removes the least useful feature based on model performance

For a house-price model, a wrapper method might start with all features, then discover that age and postal code do not improve the chosen model much, so they are removed.

Advantages:

- considers how features work with the actual model
- can produce strong performance
- can detect useful feature combinations

Limitations:

- slower than filter methods
- computationally expensive
- can overfit, especially with small data sets

## Embedded Methods

Embedded methods perform feature selection during model training.

The algorithm itself decides which features matter as part of the learning process.

Examples:

- Lasso regression
- Elastic Net
- some decision-tree and random-forest methods

In Lasso regression, less useful features can have their coefficients reduced to zero. This effectively removes them from the model.

Advantages:

- more efficient than repeatedly training many separate models
- more model-aware than simple filter methods
- reduces manual work

Limitations:

- depends on the chosen algorithm
- may still take more computation than basic filter methods
- not every model has a good embedded feature-selection mechanism

## Comparing The Three Methods

| Method | Basic idea | Strength | Weakness |
| --- | --- | --- | --- |
| [[#Filter Methods|Filter]] | Rank features using statistics | Fast and simple | May miss feature interactions |
| [[#Wrapper Methods|Wrapper]] | Try feature subsets with a model | Often strong performance | Slow and may overfit |
| [[#Embedded Methods|Embedded]] | Select features during training | Balanced and model-aware | Depends on the algorithm |

## Example: House Price Prediction

Suppose the target is house price.

Useful features might be:

- floor area
- number of bedrooms
- location
- distance to schools

Less useful or risky features might be:

- duplicate location fields
- decorative details that do not generalize
- irrelevant personal information about the owner
- features that are proxies for sensitive social factors

Feature selection should simplify the model without removing information that genuinely helps prediction.

## Ethical Link

Feature choices are not neutral.

For example, in a creditworthiness model, income might be relevant, but postcode could accidentally introduce socio-economic or racial bias. The model might appear to use geography, but actually learn patterns linked to inequality.

This is why feature selection is partly technical and partly ethical.

## Exam Pattern

If asked about the role of feature selection, say that it identifies and keeps the most informative attributes while removing irrelevant or redundant ones. This improves accuracy, reduces overfitting, speeds training, and makes the model easier to interpret.
