# A4.2.3 Dimensionality Reduction

Dimensionality reduction means reducing the number of features, variables, or attributes in a data set while keeping as much useful information as possible.

In machine learning, a **dimension** is basically a feature.

Examples:

- in customer data: age, income, location, purchase history
- in house data: floor area, bedrooms, location, age, school distance
- in an image: each pixel can act like a dimension
- in text data: word frequencies can become dimensions

Dimensionality reduction is closely related to [[A4.2.2 Feature selection|feature selection]], but it is not always identical. Feature selection chooses existing features to keep. Dimensionality reduction may also transform features into a smaller set of new combined features.

## Why Reduce Dimensions?

More features can sometimes help, but too many features can make the model worse.

Dimensionality reduction can:

- simplify the model
- reduce noise
- speed up training
- reduce memory usage
- make data easier to visualize
- reduce [[#Overfitting|overfitting]]
- help the model generalize better to unseen data

The goal is not to make the data tiny. The goal is to remove unnecessary complexity while preserving the important patterns.

## Curse Of Dimensionality

The curse of dimensionality is the set of problems that appear when data has too many dimensions compared with the number of examples.

As dimensions increase, the feature space grows very quickly. The same number of data points becomes more spread out, so patterns become harder to detect.

Simple way to think about it:

```text
few dimensions + enough examples -> patterns are easier to find
too many dimensions + not enough examples -> data becomes sparse and hard to generalize from
```

This is why adding more features is not always better.

## Data Sparsity

Data sparsity means data points are spread far apart in the feature space.

In high-dimensional data, each record can start to look almost unique. If everything looks unique, the model struggles to find general rules.

Example: for house prices, basic features like bedrooms, floor area, and location may create useful comparisons. But if hundreds of tiny details are added, such as doorknob type, curtain type, wallpaper pattern, and appliance brand, each house may look too different from every other house.

The model then has fewer meaningful near-neighbours to learn from.

## Overfitting

Overfitting happens when a model learns the training data too specifically, including noise and random details, instead of learning general patterns.

High-dimensional data increases the risk of overfitting because the model has many possible details to memorize.

An overfitted model may perform well on training data but poorly on new data.

## Main Problems Caused By Too Many Dimensions

### Higher Computational Complexity

More features usually mean more calculations. Training becomes slower and may need stronger hardware.

This connects back to [[A4.1.2 Hardware for machine learning|hardware requirements]] because large high-dimensional models may need GPUs, cloud platforms, or high-performance computing.

### Distance Metrics Become Less Useful

Some algorithms rely on distance, such as K-NN and clustering methods.

In high dimensions, many points become similarly far away from each other. This makes it harder to decide what is actually near or similar.

### Harder Visualization

Humans can understand two or three dimensions visually. High-dimensional data cannot be directly plotted in a way that is easy to interpret.

Dimensionality reduction can help convert complex data into two or three dimensions for visualization.

### More Data Is Needed

To cover a high-dimensional space properly, the amount of data needed grows very quickly.

This can become impractical because collecting, storing, cleaning, and processing enough data may be too expensive or impossible.

### Higher Memory Usage

More features require more storage and memory. This can limit whether the model can run on normal laptops, mobile devices, or edge devices.

## Example: House Price Prediction

A sensible model might use:

- number of bedrooms
- floor area
- location
- age of property
- distance to schools

An overcomplicated model might also include:

- wallpaper pattern
- type of doorknob
- brand of appliances
- curtain type
- garden plant species
- detailed maintenance history

Some of these details might occasionally matter, but many will not generalize well. They can make the model focus on accidental patterns instead of major price factors.

## How Dimensions Can Be Reduced

Common practical approaches:

- remove irrelevant features
- remove redundant features
- group similar features together
- keep only the features most strongly linked to the target
- test different feature sets and compare performance
- use statistical techniques such as PCA or LDA if appropriate

PCA and LDA are mentioned as techniques, but their internal mathematics is beyond this course.

### PCA

PCA stands for principal component analysis.

It reduces dimensions without using class labels. It is often useful for compression, visualization, and speeding up learning.

### LDA

LDA stands for linear discriminant analysis.

It uses labels and is especially useful when preparing data for [[A4.1.1 Types of machine learning#Classification|classification]] tasks, because it tries to keep the dimensions that best separate the classes.

## Dimensionality Reduction vs Feature Selection

| Idea | Meaning |
| --- | --- |
| [[A4.2.2 Feature selection|Feature selection]] | Choose which existing features to keep |
| Dimensionality reduction | Reduce the number of dimensions, sometimes by transforming features into new combined ones |

Feature selection is one way to reduce dimensions, but dimensionality reduction can be broader.

## Exam Pattern

If asked why dimensionality reduction is important, say that it reduces the number of features while preserving useful information. This simplifies the model, reduces overfitting and noise, speeds computation, reduces memory use, and helps with visualization and generalization.

If asked about the curse of dimensionality, explain that as features increase, data becomes sparse, distance measures become less meaningful, more data and memory are needed, and the model is more likely to overfit.
