# A4.3.2 Classification Techniques

Classification predicts a discrete category. It is a [[A4.1.1 Types of machine learning#Supervised Learning|supervised learning]] task because training data includes labels.

Examples:

- spam or not spam
- disease present or absent
- low, medium, or high risk
- apple or banana

This syllabus section focuses mainly on [[#K-Nearest Neighbours|K-nearest neighbours]] and [[#Decision Trees|decision trees]].

## K-Nearest Neighbours

K-nearest neighbours, or K-NN, classifies a new data point by comparing it to nearby labelled examples.

Basic process:

1. Choose a value for `k`.
2. Measure the distance between the new item and the training examples.
3. Find the `k` nearest examples.
4. Assign the class that appears most often among those neighbours.

Example: if `k = 5` and three nearest neighbours are bananas while two are apples, the new fruit is classified as banana.

## Choosing K

The value of `k` affects the model.

- small `k`: sensitive to noise and outliers
- large `k`: smoother but may ignore local patterns

Choosing `k` is a form of [[A4.3.3 Hyperparameter tuning and evaluation|hyperparameter tuning]].

## K-NN Strengths

- simple to understand
- easy to implement
- works with multi-class problems
- makes few assumptions about the data

## K-NN Weaknesses

- slow on large data sets because distances must be calculated
- sensitive to feature scaling, so [[A4.2.1 Data cleaning#Normalization|normalization]] or [[A4.2.1 Data cleaning#Standardization|standardization]] may be needed
- affected by noise, missing values, and outliers
- struggles with high-dimensional data, linking to the [[A4.2.3 Dimensionality reduction#Curse Of Dimensionality|curse of dimensionality]]

## Decision Trees

A decision tree classifies data by asking a sequence of questions about features.

Each internal node tests a feature. Each branch is an outcome of the test. Each leaf gives a final prediction.

Example for illness classification:

```text
shortness of breath?
yes -> pneumonia
no -> wheezing?
      yes -> bronchitis
      no -> common cold
```

## Decision Tree Strengths

- easy to interpret
- handles both numerical and categorical data
- does not require much scaling
- useful when explanations matter

## Decision Tree Weaknesses

- can overfit if the tree is too deep
- can become biased if some classes dominate
- simple axis-aligned splits may not capture complex boundaries

Ways to reduce overfitting include limiting tree depth, requiring minimum samples per split, pruning, or using more training data.

## K-NN vs Decision Trees

| Feature | K-NN | Decision tree |
| --- | --- | --- |
| Main idea | classify by nearby examples | classify through feature-based questions |
| Interpretability | lower for large data | usually high |
| Scaling needed | often yes | usually less important |
| Large data sets | can be slow | usually faster once trained |
| Overfitting risk | low `k` can overfit | deep trees can overfit |

## Real-World Uses

K-NN can be used in collaborative filtering recommendation systems by finding similar users or similar items.

Decision trees can be used in medical diagnosis, risk classification, fraud checks, and decision-support systems where explanations are useful.
