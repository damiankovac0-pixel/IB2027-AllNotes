# A4.3.3 Hyperparameter Tuning And Evaluation

Hyperparameters are settings chosen before training begins. They are not learned directly from the data.

Examples:

- `k` in [[A4.3.2 Classification techniques#K-Nearest Neighbours|K-NN]]
- maximum depth in a [[A4.3.2 Classification techniques#Decision Trees|decision tree]]
- learning rate in a neural network
- number of clusters in k-means

Hyperparameter tuning means trying different settings to improve model performance.

## Why Tuning Matters

Bad hyperparameters can make a good algorithm perform poorly.

Examples:

- a decision tree that is too deep may overfit
- a decision tree that is too shallow may underfit
- K-NN with `k = 1` may be too sensitive to noise
- K-NN with a very large `k` may ignore meaningful local patterns

## Confusion Matrix

A confusion matrix shows how classification predictions compare with actual labels.

For binary classification:

| Result | Meaning |
| --- | --- |
| True positive | predicted positive and actually positive |
| True negative | predicted negative and actually negative |
| False positive | predicted positive but actually negative |
| False negative | predicted negative but actually positive |

Example: in disease screening, a false negative means a sick patient is incorrectly classified as healthy.

## Accuracy

Accuracy is the proportion of predictions that are correct.

```text
accuracy = correct predictions / total predictions
```

Accuracy is useful when classes are balanced.

Weakness: accuracy can be misleading with imbalanced data. If 99% of emails are not spam, a model that always predicts not spam could appear accurate while failing at spam detection.

## Precision

Precision measures how many predicted positives were actually positive.

```text
precision = true positives / (true positives + false positives)
```

Precision matters when false positives are costly.

Example: in spam filtering, high precision helps avoid marking important emails as spam.

## Recall

Recall measures how many actual positives were found.

```text
recall = true positives / (true positives + false negatives)
```

Recall matters when false negatives are costly.

Example: in medical screening, high recall helps avoid missing sick patients.

## F1 Score

F1 score combines precision and recall.

```text
F1 = 2 * (precision * recall) / (precision + recall)
```

It is useful when there is an uneven class distribution or when both false positives and false negatives matter.

## Choosing A Metric

| Metric | Best when |
| --- | --- |
| Accuracy | classes are balanced |
| Precision | false positives are expensive |
| Recall | false negatives are expensive |
| F1 score | precision and recall both matter |

## Exam Pattern

When evaluating a supervised learning model, do not just say "accuracy" automatically. Explain why the chosen metric fits the problem. For medical diagnosis, recall may be more important than precision because missing a disease can be more harmful than a false alarm.
