# A4.3.4 Clustering

Clustering is an [[A4.1.1 Types of machine learning#Unsupervised Learning|unsupervised learning]] technique that groups data points based on similarity.

The data is unlabelled, so the algorithm is not told the correct group in advance.

## Core Idea

Objects in the same cluster should be more similar to each other than to objects in other clusters.

Examples:

- grouping customers by buying habits
- grouping traffic sensor data
- finding image regions
- discovering patterns in biological data
- segmenting social media users

## K-Means Clustering

K-means groups data into `k` clusters.

Basic process:

1. Choose the number of clusters, `k`.
2. Place initial centroids.
3. Assign each data point to the nearest centroid.
4. Recalculate centroids based on assigned points.
5. Repeat until clusters stop changing much.

Strengths:

- simple
- fast
- useful for large data sets

Weaknesses:

- must choose `k` in advance
- sensitive to outliers
- works best with roughly spherical clusters
- struggles with clusters of unusual shapes or different densities

## Spectral Clustering

Spectral clustering uses graph-based relationships between data points, rather than only direct distance to centroids.

Strength:

- can handle non-spherical or complex cluster shapes better than k-means

Weakness:

- can be computationally expensive for large data sets

## Mean Shift Clustering

Mean shift clustering finds dense regions in the data. It moves candidate centroids toward the mean of nearby points until they settle around dense areas.

Strengths:

- does not require choosing the number of clusters in advance
- can work with clusters of different shapes

Weaknesses:

- can be slower than k-means
- performance depends on the window or bandwidth size

## Real-World Applications

### Customer Segmentation

Businesses can group customers by behaviour, such as spending level, product preferences, or price sensitivity.

This can support:

- personalized marketing
- targeted recommendations
- pricing strategies
- identifying customer groups that need different services

### Traffic And Urban Planning

Traffic sensors can be clustered to find areas with similar congestion patterns. This helps planners decide where to adjust infrastructure or traffic controls.

## Exam Pattern

Clustering groups unlabelled data by similarity. It is useful when there are no predefined categories and the aim is to discover natural groups in the data.
