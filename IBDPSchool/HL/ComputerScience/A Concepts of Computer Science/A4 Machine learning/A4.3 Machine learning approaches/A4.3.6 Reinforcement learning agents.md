# A4.3.6 Reinforcement Learning Agents

Reinforcement learning is where an agent learns by interacting with an environment and receiving rewards or penalties.

This expands the overview from [[A4.1.1 Types of machine learning#Reinforcement Learning|A4.1.1 reinforcement learning]].

## Agent And Environment

An **agent** is the decision-maker.

An **environment** is the world or system the agent acts in.

Examples:

- a robotic vacuum moving around a room
- a game-playing AI in a game world
- an autonomous vehicle on roads
- a trading bot in a market environment

## Basic Cycle

```text
observe state -> choose action -> receive reward -> update policy -> repeat
```

The agent's goal is to maximize cumulative reward over time, not just the immediate reward from one action.

## State

The state is the current situation observed by the agent.

Example: for a robot vacuum, the state could include nearby obstacles, battery level, and current position.

## Action

An action is a choice the agent can make.

Example: move forward, turn left, stop, clean, or return to charger.

## Reward

A reward is feedback after an action.

Examples:

- positive reward for reaching a target
- negative reward for hitting an obstacle
- positive reward for winning a game
- negative reward for losing money in a trade

## Policy

A policy is the strategy the agent learns for choosing actions in different states.

A good policy chooses actions that lead to high cumulative reward.

## Exploration vs Exploitation

Exploration means trying new actions to learn more.

Exploitation means using the best-known action based on current knowledge.

The tradeoff matters because an agent that only explores may act randomly for too long, while an agent that only exploits may miss better strategies.

## Common Balancing Strategies

### Epsilon-Greedy

The agent usually chooses the best-known action, but sometimes chooses a random action with probability epsilon.

Epsilon may decrease over time as the agent learns.

### Upper Confidence Bound

Upper confidence bound, or UCB, chooses actions by considering both expected reward and uncertainty. It gives uncertain actions a chance if they might be valuable.

### Thompson Sampling

Thompson sampling keeps probability estimates for how good actions may be, samples from those estimates, and chooses the action with the best sampled value.

## Example: Game-Playing Agent

A game-playing agent observes the game state, chooses actions, receives rewards for progress, and penalties for losing health or failing.

Over many attempts, it learns which actions lead to higher scores or successful completion.

## Exam Pattern

In reinforcement learning, an agent observes the environment, takes actions, receives rewards or penalties, and updates its policy to maximize cumulative reward. The key challenge is balancing exploration of new actions with exploitation of known successful actions.
