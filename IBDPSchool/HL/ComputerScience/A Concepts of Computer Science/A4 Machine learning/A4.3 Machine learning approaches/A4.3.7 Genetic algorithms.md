# A4.3.7 Genetic Algorithms

Genetic algorithms are optimization techniques inspired by natural selection.

They are useful when there are many possible solutions and it is hard to search every option directly.

## Core Idea

A genetic algorithm keeps a population of possible solutions. Better solutions are more likely to be selected, combined, and changed to create the next generation.

Over time, the population should improve.

## Key Terms

### Population

The population is the set of candidate solutions.

Example: different possible school timetables or delivery routes.

### Individual

An individual is one candidate solution in the population.

### Fitness Function

A fitness function scores how good a solution is.

Example for a school timetable: the score could reward students getting first or second choices and penalize clashes or oversized classes.

### Selection

Selection chooses better individuals to become parents for the next generation.

Better fitness usually means a higher chance of selection.

### Crossover

Crossover combines parts of two parent solutions to create offspring.

Example: a timetable offspring may inherit some class blocks from one parent timetable and some from another.

### Mutation

Mutation makes small random changes to a solution.

Mutation helps maintain diversity and prevents the algorithm from getting stuck too early.

### Evaluation

Evaluation applies the fitness function to the new solutions.

### Termination

The algorithm stops when a condition is met, such as:

- fixed number of generations reached
- good enough solution found
- no significant improvement for several generations

## Process

```text
create population -> evaluate fitness -> select parents -> crossover -> mutate -> evaluate offspring -> repeat
```

## Real-World Applications

Genetic algorithms can be used for:

- timetable scheduling
- route optimization
- wind farm layout
- vehicle design
- resource allocation
- complex engineering optimization

## Strengths

- useful for complex search spaces
- can find good solutions without testing every possibility
- works when there is no simple formula for the best solution
- can handle multiple constraints

## Weaknesses

- may not find the perfect solution
- can be computationally expensive
- depends heavily on the fitness function
- poor parameter choices can lead to weak results

## Exam Pattern

For a scenario, identify the population, fitness function, selection, crossover, mutation, and stopping condition. The fitness function is usually the most important part to explain because it defines what "good" means.
