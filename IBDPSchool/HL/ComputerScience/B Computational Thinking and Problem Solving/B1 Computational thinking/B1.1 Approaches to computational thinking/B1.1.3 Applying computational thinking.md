# B1.1.3 Applying Computational Thinking

Computational thinking is used in many areas of computer science. The same four concepts — decomposition, pattern recognition, abstraction, algorithmic design — apply across domains.

## Software Development

When creating a large customer relationship management (CRM) system:

- **Decomposition**: split the project into modules such as lead tracking, customer management, and communication tools.
- **Pattern recognition**: identify common software bugs from past projects to speed up debugging.
- **Abstraction**: use object-oriented programming to hide complex operations inside objects with clean interfaces.
- **Algorithmic design**: implement precise logic for tasks such as automatic email scheduling based on user behaviour.

## Data Analysis

Analysing large datasets to generate business insights:

- **Decomposition**: break analysis into data collection, cleaning, exploration, modelling, and interpretation.
- **Pattern recognition**: identify trends, correlations, and outliers in historical data.
- **Abstraction**: define tables and relationships that represent real-world entities while ignoring irrelevant details.
- **Algorithmic design**: develop queries and statistical methods to extract and summarise data efficiently.

## Network Security

Protecting networks from cyber threats:

- **Decomposition**: separate security into physical security, network infrastructure, and application security.
- **Pattern recognition**: identify attack patterns such as DDoS, phishing, and port scanning.
- **Abstraction**: use standard security models and protocols that define rules without specifying low-level details.
- **Algorithmic design**: create encryption algorithms and protocols such as SSL and TLS.

## Game Development

Creating an intelligent non-playing character (NPC):

- **Decomposition**: split into sensing, decision-making, actions, and pathfinding.
- **Pattern recognition**: notice that certain player behaviours tend to repeat (e.g. rushing, camping).
- **Abstraction**: model NPC state using a finite state machine with states such as patrol, chase, and attack, ignoring irrelevant environment details.
- **Algorithmic design**: write the step-by-step logic for the NPC to transition between states based on conditions.

## Machine Learning

Building and training a machine learning model:

- **Decomposition**: break the process into problem definition, data collection, preprocessing, model selection, training, evaluation, and deployment.
- **Pattern recognition**: identify features and relationships in the training data.
- **Abstraction**: select which features to include and which to discard; create a simplified representation of the problem.
- **Algorithmic design**: choose and tune the learning algorithm, define the loss function, and set the training procedure.
