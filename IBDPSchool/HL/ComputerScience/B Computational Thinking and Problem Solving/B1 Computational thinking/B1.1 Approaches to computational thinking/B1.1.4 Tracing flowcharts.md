# B1.1.4 Tracing Flowcharts

A flowchart is a diagram that represents a process, algorithm, or program using standard symbols connected by arrows.

Flowcharts help visualise the logical flow of a solution before coding.

## Standard Symbols

| Shape | Name | Meaning |
| --- | --- | --- |
| Rounded rectangle | Terminator | Start or end of the process |
| Parallelogram | Input / output | Read input or display output |
| Rectangle | Process | Action, calculation, or assignment |
| Diamond | Decision | True / false or yes / no condition |
| Arrow | Flowline | Direction of data flow |
| Small circle | Connector | Continuation across pages |

## How To Trace A Flowchart

To trace a flowchart, follow the flowlines step by step and record the value of each variable as it changes.

Keep a trace table with columns for each variable, decision outcomes, and output.

Example: find the larger of two numbers.

```text
START
INPUT a, b
IF a > b THEN
    max = a
ELSE
    max = b
OUTPUT "The larger value is ", max
END
```

Trace table for input 7 and 3:

| a | b | a > b? | max | output |
| 7 | 3 | TRUE | 7 | The larger value is 7 |

## Labelling Decisions

Always label the branches of a decision box. Without labels, the reader cannot tell which path is taken when the condition is true and which when it is false.

Use "Yes" / "True" and "No" / "False".

## Counting Iterations

When a flowchart contains a loop, you may be asked how many iterations run before the program ends.

Example:

```text
A = 10
B = 100
REPEAT
    A = A + 10
    B = B - 10
UNTIL A > B
```

| Iteration | A | B | A > B? |
| 1 | 20 | 90 | FALSE |
| 2 | 30 | 80 | FALSE |
| 3 | 40 | 70 | FALSE |
| 4 | 50 | 60 | FALSE |
| 5 | 60 | 50 | TRUE → end |

Five iterations.

## Linear Search Flowchart

A linear search checks every element in an array from the start until the target is found.

```text
START
SET target = desired value
SET index = 0
WHILE index < array.length
    IF array[index] == target THEN
        OUTPUT "Found at position ", index
        END
    index = index + 1
OUTPUT "Not found"
END
```

## Common Mistakes

- not labelling decision branches
- flowlines disconnected from shapes
- missing start or end symbols
- placing output inside a process shape instead of using the input/output symbol
- forgetting to update variable values in a trace table
- multiple flowlines entering a process from different directions without a connector

## Exam Pattern

You may be given a flowchart and asked to:
- trace it with given input values
- state the output
- determine the purpose of the algorithm
- count the number of iterations
- identify errors or edge cases

Use a trace table and record each variable change row by row.
